## Permission evidence rules
Role and permission claims must be grounded in code, policy, backend checks, database rules, auth provider configuration, route guards, UI gating, or tests.

For each role, include:
- role identifier
- display label
- visible menus/routes
- allowed actions
- denied or hidden actions
- approval or admin powers
- data scope
- tenant/workspace/org constraints
- source evidence
- confidence level

If UI gates and backend permissions disagree, document the conflict and record it in `./DOC_GENERATION_REPORT.md`.
