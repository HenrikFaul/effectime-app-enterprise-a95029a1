## Feature extraction rules
Each feature must have a stable feature ID and evidence-backed scope.

Required fields:
- feature ID
- feature name
- business purpose
- user roles
- UI entry points
- routes or screens
- backend handlers/services
- entities
- validations
- feature flags
- related tests
- related process flows
- confidence level

Do not create a feature entry from naming alone. A feature must map to behavior, UI, backend, data, tests, or documented historical evidence.

## Repository-relative rule
Use repository-relative evidence paths only.
