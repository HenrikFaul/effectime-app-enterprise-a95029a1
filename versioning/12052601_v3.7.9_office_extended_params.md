# v3.7.9 — Extended Office Parameters

**Date:** 2026-05-12  
**Branch:** `claude/create-software-documentation-O7kj1`  
**Type:** Feature (additive, non-breaking)

## Summary

Extended the office/location management with rich parameters: contact info, opening hours, equipment/facilities inventory, and permanent minimum staffing requirements.

## DB Changes

### `enterprise_offices` — new columns
| Column | Type | Notes |
|---|---|---|
| `email` | TEXT | nullable |
| `phone` | TEXT | nullable |
| `manager_name` | TEXT | nullable |
| `deputy_name` | TEXT | nullable |
| `opening_hours` | JSONB | `{"1": {"open":"08:00","close":"17:00","closed":false}, …}` keys 1–7 (Mon–Sun) |

### `enterprise_office_equipment` — new table
- id, workspace_id, office_id (FK ON DELETE CASCADE)
- name (required), description, quantity (≥1)
- `required_skill_id` → `enterprise_skills` (nullable, ON DELETE SET NULL)
- RLS: members SELECT, admins INSERT/UPDATE/DELETE

### `enterprise_office_min_staffing` — new table
- id, workspace_id, office_id (FK ON DELETE CASCADE)
- `business_role` TEXT (nullable), `skill_id` → `enterprise_skills` (nullable)
- `min_headcount` INTEGER ≥ 1
- CONSTRAINT: business_role IS NOT NULL OR skill_id IS NOT NULL
- RLS: members SELECT, admins INSERT/UPDATE/DELETE

## UI Changes

`src/components/enterprise/OfficeManager.tsx` — full redesign:
- Compact list view: shows phone, email, manager as chips
- Dialog editor with 4 collapsible sections
- Opening hours grid: time pickers per day + Closed toggle (Switch)
- Equipment: live CRUD inline in dialog (add on-the-fly, delete)
- Min staffing: live CRUD inline in dialog (role + skill dropdowns)
- Existing offices: equipment and staffing load immediately when dialog opens
- New offices: note shown, equipment/staffing available after first save

## i18n

`office_mgr.*` namespace extended with 40+ new keys across all 5 locales.

## Quality
- TypeScript: 0 errors
- No regressions to existing office/coverage-rule/member-priority features
