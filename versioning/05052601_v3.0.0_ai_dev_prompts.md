### Prompt 1.3 — `profiles.preferred_locale` persistence

Migration `supabase/migrations/<datestamp>_add_preferred_locale.sql`:

```sql
alter table public.profiles
  add column if not exists preferred_locale text;

alter table public.profiles
  add constraint profiles_preferred_locale_check
  check (preferred_locale is null or preferred_locale in ('en','hu'));
```

- Frontend: when the user changes language while signed in, also `update profiles set preferred_locale = ?` for the current user.
- On sign-in, hydrate `localStorage.effectime.locale` from `profiles.preferred_locale` if set.
- Do not block sign-in on this update; failures log and degrade silently.

Acceptance:

- New column visible via Supabase types regen.
- RLS unchanged on `profiles`; no new policy required (user already self-updates own profile per existing rules).
- Migration is idempotent (`if not exists`, named constraint).
