## Implementation details

### conflictEngine.ts fallback messages
The `message` field on `ConflictResult` is documented as a "Hungarian fallback — kept for backward compatibility". The UI uses `formatConflict(c, t)` from `conflictEngineI18n.ts` for all named conflict codes, so these strings only appear in logs, toasts for unknown codes, and VALIDATION_ERROR passthrough. Replacing them with English makes debug output internationally readable without any UI impact.

### ApprovalInbox stale closure
`fetchRequests` used `memberInfo` state at lines 98–100 for the team/role filter step. `memberInfo` is set via `setMemberInfo(mMap)` earlier in the same function invocation, but React's `setState` is asynchronous — the state has not re-rendered when the filter runs. Fix: hoist `const mMap` before the `if (userIds.length > 0)` block and reference `mMap` directly in the filter step.

### superadmin-hub pagination
`auth.admin.listUsers({ perPage: 10000 })` silently caps at 1000 (Supabase maximum). Platform-overview user counts and list-workspaces owner email resolution now loop with `perPage: 1000` until the batch is smaller than 1000 (last page). The list-workspaces loop also short-circuits once all owner IDs are resolved to avoid fetching unnecessary pages.

### ms365-sync OAuth upsert
The upsert at the end of `handleCallback` did not capture its return value, so DB errors (e.g. a missing unique constraint on `workspace_id, user_id, provider`) would silently succeed from the caller's perspective, leaving the integration unregistered. Now destructures `{ error: upsertErr }` and throws, which is caught by the surrounding try/catch and returns a 500 HTML error page.

## ATS candidate pipeline schema (migration 20260515080000)

The `useCandidates.ts` hook references `candidates`, `interview_slots`, and three RPCs that previously had no backing schema. This migration provisions them all.

**`candidates`**: workspace-scoped candidate store with status enum check, optional `enterprise_membership_id` FK (set when hired candidate becomes a member), metadata jsonb, and `updated_at` trigger.

**`interview_slots`**: interview scheduling with `interviewer_membership_ids[]`, unique `booking_token` (consumed on self-booking), outcome rating/recommendation checks, and `updated_at` trigger.

**RLS**: authenticated members SELECT both tables; owners/resourceAssistants have full management. `anon` can SELECT available `interview_slots` by booking token (public self-booking page).

**`candidate_create_slot`** (SECURITY DEFINER): verifies role → validates time order → calls `candidate_interview_slot_eligible` (leave + double-booking check) → inserts slot with generated booking token. Returns slot UUID.

**`candidate_self_book`** (SECURITY DEFINER, anon + authenticated): atomic self-booking via token — FOR UPDATE lock prevents race, upserts candidate by email, advances status to `interview`, books slot, nulls token. Returns jsonb `{ok, slot_id, slot_start, slot_end, candidate_id}`.

**`candidate_generate_onboarding`** (SECURITY DEFINER): finds latest published template → inserts `enterprise_onboarding_instances` with `due_at = _start_date + 30 days`. Returns null instance gracefully when no membership or template exists.

Migration applied to live Effectime project (oezlzzmzzvbvinuysxaz).
