## Implementation details

### conflictEngine.ts fallback messages
The `message` field on `ConflictResult` is documented as a "Hungarian fallback â€” kept for backward compatibility". The UI uses `formatConflict(c, t)` from `conflictEngineI18n.ts` for all named conflict codes, so these strings only appear in logs, toasts for unknown codes, and VALIDATION_ERROR passthrough. Replacing them with English makes debug output internationally readable without any UI impact.

### ApprovalInbox stale closure
`fetchRequests` used `memberInfo` state at lines 98â€“100 for the team/role filter step. `memberInfo` is set via `setMemberInfo(mMap)` earlier in the same function invocation, but React's `setState` is asynchronous â€” the state has not re-rendered when the filter runs. Fix: hoist `const mMap` before the `if (userIds.length > 0)` block and reference `mMap` directly in the filter step.

### superadmin-hub pagination
`auth.admin.listUsers({ perPage: 10000 })` silently caps at 1000 (Supabase maximum). Platform-overview user counts and list-workspaces owner email resolution now loop with `perPage: 1000` until the batch is smaller than 1000 (last page). The list-workspaces loop also short-circuits once all owner IDs are resolved to avoid fetching unnecessary pages.

### ms365-sync OAuth upsert
The upsert at the end of `handleCallback` did not capture its return value, so DB errors (e.g. a missing unique constraint on `workspace_id, user_id, provider`) would silently succeed from the caller's perspective, leaving the integration unregistered. Now destructures `{ error: upsertErr }` and throws, which is caught by the surrounding try/catch and returns a 500 HTML error page.
