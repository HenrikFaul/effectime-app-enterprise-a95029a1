# Syncfolk AI dev prompts — Enterprise Phase 2: Leave Request & Approval Workflow

**Versioning ID:** 16041102  
**Version:** v2.1.0

## Jira ticketek ebben a fázisban
- **SYN-165** — Enterprise workflow state tables (request states, decision states, transitions)
- **SYN-150** — Approval inbox, filters and bulk decisions
- **SYN-149** — Leave request submission and management

## Összefoglaló
Enterprise B2B modul 2. fázis: Távolléti kérelem beküldés és jóváhagyási workflow.

### Implementált funkciók

1. **Leave Request DB schema** (SYN-165, SYN-149):
   - `leave_requests` tábla: workspace_id, user_id, leave_type, start_date, end_date, status, comment, reviewer_id, reviewed_at, review_comment
   - `approval_decisions` tábla: leave_request_id, workspace_id, decided_by, decision, comment
   - Enumok: `leave_request_status` (draft/pending/approved/rejected/cancelled/expired), `leave_type` (vacation/sick_leave/unpaid_leave/other)
   - RLS: tagok saját kérelmet adhatnak be, owner/resourceAssistant update jogosultság, owner delete jogosultság

2. **Leave Request submission UI** (SYN-149):
   - `LeaveRequestDialog`: dátumválasztó (start/end), típus választó, megjegyzés mező
   - `LeaveRequestList`: kérelmek listázása workspace-en belül, visszavonás funkció saját pending kérelmekre
   - Profil nevek lekérése a user_id alapján

3. **Approval Inbox** (SYN-150):
   - `ApprovalInbox`: szűrők (státusz: összes/függőben/jóváhagyva/elutasítva/visszavonva)
   - Egyéni jóváhagyás/elutasítás megjegyzéssel
   - Bulk kijelölés és tömeges jóváhagyás/elutasítás
   - Döntés audit rekord mentése az `approval_decisions` táblába

4. **WorkspaceDashboard bővítés**:
   - Új tabok: "Kérelmek" (FileText ikon), "Jóváhagyások" (CheckSquare ikon, csak admin)
   - TabsList flex-wrap a reszponzivitáshoz
