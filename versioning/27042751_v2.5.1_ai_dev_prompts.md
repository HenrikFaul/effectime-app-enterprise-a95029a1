## Business request
A new calendar/capacity update caused regressions in previously working enterprise calendar functionality. Fix the regressions without duplicating existing features and without breaking the already delivered multi-position / multi-skill coverage work.

## Required fixes
1. Telephelyi szabály mentés
   - Fix the save error: `Could not find the 'business_roles' column of 'enterprise_office_coverage_rules' in the schema cache`.
   - Keep the new multi-position / multi-skill payload as the primary path when the DB supports it.
   - Add a schema-cache-safe fallback to the legacy columns (`business_role`, `skill_id`) only when the missing-column error specifically concerns `business_roles` or `skill_ids`.
   - Do not swallow unrelated database errors.

2. Kapacitástervező navigation
   - Remove the `Szabályok szerkesztése` shortcut from `CoveragePlannerView` because it navigates to the wrong place.
   - Do not remove the actual rule-management functionality elsewhere.

3. Kapacitástervező monthly grid
   - In monthly mode, do not render the long `Nincs szabály — adj hozzá...` text inside the day area.
   - Render quiet grey day cells instead.
   - Keep thin grey separators between days so the month does not visually merge into one block.
   - Avoid dynamic Tailwind classes such as `col-span-${colCount}` for variable column spans.

4. Timeline layout
   - Move the filter panel into a narrow left sidebar on desktop.
   - Keep the calendar grid next to the filter panel.
   - Keep mobile responsive behavior stacked vertically.

## Regression constraints
