## Design choice: template-first, AI-second

The strategy doc framed this as an "AI Document Generator". A pure-AI
implementation would require ANTHROPIC_API_KEY to be configured before
the feature works at all — too brittle. Instead:

- **Pure template substitution** is the always-working core path. 5
  system templates ship out of the box.
- **AI polish** is an optional enhancement gated on `ANTHROPIC_API_KEY`
  being present in Supabase function secrets. If unset, the edge fn
  returns the original HTML unchanged with a `hint` explaining how to
  enable it. UX shows a non-error toast.

This keeps the feature working on day one for every workspace and lets
the platform admin opt into AI when they're ready.

## DB

- `document_templates` — id, workspace_id (nullable for system),
  doc_type, language, body_html, variables[], is_system.
- `generated_documents` — append-only history with status.
- `document_substitute(body_html, vars)` — IMMUTABLE token replace.
- `document_generate(...)` — SECURITY DEFINER RPC; auto-populates
  workspace + member context.

## Edge function

`document-ai-polish` (v1, deployed) — calls Claude Haiku 4.5 with a
"polish this HR doc, preserve HTML structure" system prompt. Falls back
to unchanged content + `ai_available: false` if the key isn't set.

## UI

`DocumentGeneratorPanel`: template picker, extras JSON input (for
context variables like `leave.start_date`), Generate + Polish-with-AI
buttons, HTML preview pane, recent-documents list. Wired inside the
Reports tab.
