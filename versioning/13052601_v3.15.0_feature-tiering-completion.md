# v3.15.0 — Feature Tiering completion (gating, audit, i18n, demo seed, tests)

**Date:** 2026-05-13
**Branch:** `claude/review-project-requirements-lR0Wp`
**Scope:** Close the remaining gaps in the v3.13.x tiering system delivery so the
contract is end-to-end functional, not theatre.

## Background

The previous tiering work (v3.13.x) delivered the feature catalog (135 features,
3 tiers, 5 addons), the FeatureTiersTab admin UI, tier-aware workspace creation
RPC, and a `useFeature` hook — but a deeper audit found that several spec
requirements were either dead code, broken, or never wired up. This release
closes those gaps.

## What changed

### 1. FeatureGate actually wired up (was: dead code)

The `<FeatureGate>` component and `useFeature` hook existed but were never
used. They now gate:

- `src/components/shell/WorkspaceSidebar.tsx` — tier-aware nav filtering on top
  of existing role permissions. Each sidebar item carries a `featureKey`
  representing the section (members → `members_list`, calendar →
  `calendar_monthly`, settings → `ws_general`, etc.).
- `src/components/enterprise/WorkspaceDashboard.tsx` — premium admin tabs
  (Analytics → `executive_dashboard`, Developer → `open_api`, Security →
  `soc2_iso`) are now wrapped in `<FeatureGate>` with a `LockedFeatureNotice`
  fallback.

**Fail-open behavior:** FeatureGate and the sidebar gate are intentionally
permissive when the workspace returns zero enabled features (= legacy
workspace with no tenant binding yet). Once any feature is enabled, both
enforce strictly. Override with `strictWhenEmpty` on FeatureGate.

### 2. Localization (was: 22+ hardcoded Hungarian strings)

CLAUDE.md mandates every user-facing string be in `src/i18n/resources/*`.

- `FeatureTiersTab.tsx` — 22 hardcoded strings moved to a new `feature_tiers.*`
  namespace.
- `CreateWorkspaceDialog.tsx` — 3 tier-related strings moved to
  `create_workspace.tier_*`.
- New `feature_gate.locked_title` / `feature_gate.locked_desc` for the
  LockedFeatureNotice fallback.
- Keys added in **all 5 locales** (en, hu, cs, sk, pl). i18n parity test
  passes (146/146 total tests green).

### 3. Audit trail on tier/addon changes (was: missing)

Spec phase 8 mandates an audit trail for tier/addon changes — there was none.

- New table `platform_audit_events` (id, actor_id, action, target_type,
  target_id, prev_state, new_state, metadata, created_at) with RLS that only
  allows platform admins to read/write and prohibits UPDATE/DELETE (immutable
  audit log).
- `FeatureTiersTab` now writes an audit row on every `tier_feature_enabled`,
  `tier_feature_disabled`, `addon_feature_enabled`, `addon_feature_disabled`,
  and `feature_routing_updated` action.

### 4. Tier-aware demo seeding (was: tier_key ignored after RPC)

`seed-demo-workspace` accepted `tier_key` but ignored it for sizing — every
demo got the same enterprise-sized dataset.

- New `TIER_SEED_OVERRIDES` export in `supabase/functions/seed-demo-workspace/seed-data.ts`.
- Freemium → 6 members, 1 team, no projects/agile/scenarios.
- Pro → 12 members, 3 teams, 2 projects, 12 agile issues.
- Enterprise → unchanged (full catalog).
- Owner-specific config in `enterprise_seed_config` still wins so operators can
  override per workspace.

### 5. CSV docs populated from source-of-truth (was: empty stubs)

`docs/tiering/dependency_matrix.csv`, `features.csv`, and `tiers_matrix.csv`
were headers-only. New `scripts/build_tiering_csvs.mjs` parses every features
INSERT + tier/addon binding INSERT + post-hoc dependency UPDATE across the
migration history and regenerates the three CSVs. Run after seed changes to
keep docs in sync.

### 6. Tier behavior tests (was: missing)

New `src/test/featureTiering.test.ts` (13 tests). Covers:

- Catalog invariants (≥100 features, unique keys).
- Every dependency references a known feature.
- No dependency cycles (caught a real 2-cycle: `leave_conflict_check ↔
  leave_daily_rules` — fixed below).
- Tier inheritance: freemium ⊆ pro ⊆ enterprise.
- Demo seed sizing is monotonic across tiers and disables agile content on
  freemium.
- `useFeature` hook contract: empty set → everything disabled.

### 7. Seed data bug fixes (caught by the new tests)

The tests caught real seed bugs that the developer's manual review missed:

- Four features had multi-deps stored as a single comma-joined string
  (`ARRAY['offices,attendance_log']` instead of `ARRAY['offices','attendance_log']`):
  `site_assignment`, `ai_smart_schedule`, `ai_burnout_predict`, `burnout_engine`.
  This silently broke their dependency resolution in `tenant_enabled_features`
  and the FeatureTiersTab missing-prereq badge.
- `leave_conflict_check` ↔ `leave_daily_rules` formed a 2-cycle. Resolved by
  dropping the upward dependency from `leave_daily_rules` (the rules are the
  source-of-truth used *by* the conflict check, not vice versa).

Both fixed in `supabase/migrations/20260513120100_fix_feature_dependencies.sql`.

## Known limitations / next steps

- FeatureGate integration is currently scoped to the workspace sidebar and the
  three highest-value premium tabs. Deeper module-level gating (e.g. inside
  individual settings tabs, dialogs, and quick-action buttons) is intentionally
  deferred — the tier system needs to bake in production before expanding the
  gate surface area.
- `platform_audit_events` is not yet surfaced in the Superadmin UI; that's a
  future v3.15.x task (audit log viewer with filters).
- The CSV builder treats migrations as the source of truth and applies
  `UPDATE features SET dependencies` statements, but does not yet model
  arbitrary post-seed mutations. Adequate for the current corrective patch.
