## Changes

### `src/index.css` — additive polish layer (lines 405+)

All changes are append-only, scoped to `@layer base` and `@layer utilities`.
No existing rule was modified.

- **Ambient body backdrop** — three soft radial gradients (primary + accent
  tinted) fixed to the viewport. Gives every in-app screen a premium,
  depth-aware background instead of flat off-white. Respects dark mode via
  the existing HSL tokens.
- **Premium focus-visible ring** — WCAG 2.2 SC 2.4.7 compliant 3:1 contrast
  halo (2px background spacer + 4px primary ring + 8px primary glow) on
  every interactive element. Replaces the browser default 1px dotted outline.
- **Refined scrollbars** — thin (10px), token-coloured thumb, hover
  highlights to `primary/55`. WebKit + Firefox. Matches the rest of the
  design language.
- **Tabular figures everywhere** — `font-variant-numeric: tabular-nums`
  globally on `table`, `[data-numeric]`, `.font-mono`. Aligns KPI numbers
  in the reports dashboard and member lists without changing markup.
- **Selection accent** — `::selection` uses `primary/25`.
- **New opt-in utilities** (not auto-applied, available for future use):
  - `.kpi-pop` — soft 3px lift + elevated shadow on hover.
  - `.kpi-shine` — animated gradient sweep for big KPI numbers.
  - `.glass-header` — backdrop-blur sticky header surface.
- **`prefers-reduced-motion`** — all new animations honoured.

## Anti-regression analysis

- Zero component file edits → zero render-tree change → zero behavioural risk.
- All new CSS rules are either (a) on `body` / `*:focus-visible` / `::selection`
  (UA defaults being replaced — safe) or (b) opt-in utility classes that
  nothing currently consumes.
- `prefers-reduced-motion` is honoured for every new animation.
- `index.html` `<meta keywords>` is a head-only signal; no other crawler or
  runtime path is affected.
