## Three pillars (per the strategy doc)

1. **In-app onboarding checklist** — 7 whitelisted items
   (`team_invited`, `sites_configured`, `schedule_template_created`,
   `calendar_connected`, `first_schedule_published`,
   `mobile_app_installed`, `member_skill_added`). Floating widget that
   self-hides at 100% completion, persisted per-workspace in
   localStorage. Click-to-jump wired to dashboard tabs.

2. **NPS automation** — 0–10 score scale + contextual follow-up
   (different prompts for detractors / passive / promoters). 60-day
   dedup so a user never gets two surveys in close succession. Floating
   bottom-right banner; opt-out per session.

3. **Health scoring** — 5-component weighted score (0–100):
   `onboarding` (0–25) + `team_size` (0–25) + `activity` (0–25)
   + `features` (0–15) + `support` (0–10). Trend (improving/stable/declining)
   computed by comparing to previous snapshot.

## DB structure

| Object | Purpose |
|---|---|
| `customer_success_onboarding_progress` | per-workspace item completion log |
| `customer_success_nps_surveys` | per-user surveys + response data |
| `customer_success_health_scores` | per-workspace snapshots |
| `customer_success_record_onboarding_step(workspace_id, item_key)` | SECURITY DEFINER, writes to onboarding_progress |
| `customer_success_trigger_nps(workspace_id, category)` | SECURITY DEFINER, dedup-aware survey creator |
| `customer_success_submit_nps(survey_id, score, feedback)` | SECURITY DEFINER, owner-only update |
| `customer_success_calculate_health_score(workspace_id)` | SECURITY DEFINER, multi-component scorer |

All 3 tables policy-block direct client INSERT — the 4 RPCs are the
sole writers. Reads scoped to member / owner-RA / platform admin per
table.

## Tier mapping

All 3 feature_keys (`cs_onboarding_checklist`, `cs_health_score`,
`cs_nps_survey`) mapped to **all 3 tiers** (Freemium + Pro + Enterprise).
Customer-success retention matters as much for free as for paid — and
Freemium users who complete onboarding are the highest-converting
upgrade segment.
