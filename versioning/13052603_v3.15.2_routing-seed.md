# v3.15.2 — Routing seed for all 135 features

**Date:** 2026-05-13
**Branch:** `claude/review-project-requirements-lR0Wp` (PR #73)
**Scope:** Populate the previously-empty `features.route_path` and
`features.menu_path` columns for all 135 features so the FeatureTiersTab
routing audit banner stops reporting "Hiányzó menu_path 135".

## Background

`features.route_path` and `features.menu_path` columns were added in
v3.13's `20260512231811_*.sql` migration but never seeded. The
FeatureTiersTab routing tree and audit banner relied on them; the
"Útvonal / Menü" tab showed every feature with red "nincs route" /
"nincs menü" badges, and the audit banner flagged all 135 as missing.

Verified the gap by querying the remote DB:

```sql
SELECT count(*) AS total,
       count(*) FILTER (WHERE route_path IS NOT NULL AND route_path <> '') AS with_route
FROM public.features;
-- → total=135, with_route=0
```

## What changed

### Migration `20260513140000_seed_feature_routes_menus.sql`

135 `UPDATE features SET route_path, menu_path` statements covering every
feature_key in the catalog. Route conventions:

| Domain | Route prefix |
|--------|--------------|
| In-workspace features | `/app/<top-tab>[/<sub-tab>]` (e.g. `/app/calendar`, `/app/resources/agile/kanban`) |
| Platform / superadmin | `/superadmin/*` (e.g. `/superadmin/tiers`, `/superadmin/audit`) |
| Pre-workspace | `/auth`, `/profile`, `/unsubscribe`, `/reset-password` |
| Admin tools | `/admin`, `/admin/api` |

Menu breadcrumbs are Hungarian (workspace primary language), e.g.:

- `members_list` → `['Tagok', 'Listázás']`
- `calendar_monthly` → `['Naptár', 'Havi nézet']`
- `payroll_export` → `['Munkaóra', 'Bérszámfejtés export']`
- `superadmin_control_plane` → `['Superadmin', 'Control Plane']`

The migration ends with a `DO $$ ... $$` sanity guard that emits a
PostgreSQL WARNING if any feature row is still missing `route_path` or
`menu_path` after the updates — so a future feature added without a
matching UPDATE in this file gets flagged immediately at migration time
instead of regressing the audit banner silently.

### Applied to remote DB

Migration applied via Supabase MCP `apply_migration` against project
`oezlzzmzzvbvinuysxaz`. Post-check:

```
total=135, with_route=135, with_menu=135, missing_route=0, missing_menu=0
```

### CSV / JSON regen

`scripts/build_tiering_csvs.mjs` extended to parse `UPDATE features SET
route_path = …, menu_path = ARRAY[…]` statements from migrations
(previously only `dependencies` UPDATEs were honored). The script now:

- Adds `route_path` + `menu_path` to `docs/tiering/features.csv`
  (2 new columns; menu_path joined with ` > ` for readability).
- Adds `route_path` + `menu_path` fields to each entry in
  `docs/tiering/features.json`.

Both regenerated; 135/135 features have both fields.

## Deduplication notes

Without care, two features could share the same `(route_path, menu_path)`
combo and trip the audit banner's "Duplikált route_path + menu_path"
section. Caught one such collision during seed authoring:

- `workspace_general_settings` and `ws_general` both naturally pointed at
  `/app/settings/general` with menu `Beállítások > Általános`.
- Resolved by routing `workspace_general_settings` to a sub-page
  `/app/settings/general/regional` with menu `Beállítások > Általános >
  Régió & időzóna` (which actually matches its description "TZ/locale-
  only settings").

Confirmed 0 duplicate combos in the final mapping.

## Known limitations

- Routes are *conceptual* — the app's actual HashRouter only resolves
  top-level pages (`/app`, `/superadmin`, etc.), and within `/app` the
  navigation is tab-based. The `route_path` reflects where each feature
  *conceptually* lives in the navigation tree, which is what the
  FeatureTiersTab routing tree and the audit banner care about. If a
  later refactor splits `/app` into real routes per top-tab, these
  values become "ground truth" rather than "design intent" — no
  re-seeding needed.
- Menu segments are HU; cs/sk/pl users see them in HU. Per
  `docs/tiering/docs_tasks.md`, full per-segment localization is a
  backlog item.
