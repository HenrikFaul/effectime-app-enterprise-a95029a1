# v3.41.6 — Fix: Open Shift Position Selector

**Date:** 2026-05-17
**Branch:** claude/analyze-repo-structure-dE8F4
**Type:** bugfix

## Root Cause

`OpenShiftManager` used `PositionPickerDialog` for position selection. That dialog reads only from
`enterprise_workspace_roles` (structured catalog positions). Manually-added positions live as free-text
`business_role` strings on `enterprise_memberships` / `enterprise_member_role_allocations` and were
never included — so "Optometrista" and other hand-entered positions were invisible in the open shift form.

`OfficeCoverageRuleManager` correctly unions all three sources, so it showed "Optometrista" fine.

## Fix

- Created `src/hooks/useWorkspaceAllPositions.ts`: fetches and deduplicates positions from
  `enterprise_workspace_roles.name` + `enterprise_memberships.business_role` +
  `enterprise_member_role_allocations.business_role`.
- Replaced `PositionPickerDialog` in `OpenShiftManager` (both compact and full modes) with a
  flat `Select` dropdown powered by `useWorkspaceAllPositions`. Matches the UX pattern of
  `OfficeCoverageRuleManager`.

## Files Changed

- `src/hooks/useWorkspaceAllPositions.ts` — new unified positions hook
- `src/components/enterprise/calendar/OpenShiftManager.tsx` — position picker replaced with Select
