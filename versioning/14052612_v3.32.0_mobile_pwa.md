# v3.32.0 — Mobile PWA + Offline scaffold (Top-20 Rank 7)

**Date:** 2026-05-14
**Status before:** MISSING (green-field, largest scope).
**Status after:** DONE for installable PWA + basic offline; full Workbox + IndexedDB sync + FCM deferred.

## What ships

- `public/manifest.webmanifest` — full PWA manifest with icons,
  shortcuts (Clock in / Leave request), theme color.
- `public/sw.js` — hand-written Service Worker:
  - CacheFirst for static assets.
  - NetworkFirst with cache fallback for navigation.
  - Bypass for Supabase auth/edge/realtime (live sessions never cached).
- `src/lib/pwa/registerSW.ts` — registers /sw.js at boot (skips on localhost).
- `src/components/pwa/InstallPwaPrompt.tsx` — floating Install banner
  with 30-day dismissal memory.
- `index.html` updated: manifest link + theme-color + Apple PWA meta tags.
- `src/main.tsx` calls `registerEffectimeServiceWorker()` + `captureInstallPrompt()` at startup.
- 15 localized strings × 5 locales (`pwa.*` namespace).

## Why hand-written SW

Adding `vite-plugin-pwa` modifies the Vite build pipeline; out of scope
for a single session. The hand-written SW gives us:

- Installability (Add to Home Screen works).
- Basic offline (app shell + last-visited pages reachable when offline).
- Bypass for live Supabase traffic (correctness preserved).

## Deferred (v3.32.1+)

- `vite-plugin-pwa` integration + Workbox auto-precache manifest from build output.
- IndexedDB write queue for offline submissions (leave requests, clock-in events).
- Firebase Cloud Messaging push notifications.
- Capacitor native plugins (NFC, biometric auth) — Capacitor itself is already in package.json.
