# v3.9.0 — Burnout & Wellbeing Detection Engine

**Date:** 2026-05-12
**Branch:** claude/create-software-documentation-O7kj1
**Type:** New feature

## Scope

Automated wellbeing scoring engine with manager dashboard and employee self-view card.

## Components delivered

### `src/components/enterprise/wellbeing/WellbeingDashboard.tsx`
- Props: `{ workspaceId: string; isAdmin: boolean }`
- `calculateWellbeingScore()`: 5-component weighted algorithm
  - Overtime density: 30%
  - Leave utilization: 20%
  - Weekend work density: 25%
  - Schedule stability: 15%
  - Recovery time: 10%
- Score range 0–100; tiers: Healthy (≥70), Monitor (40–69), At Risk (<40)
- "Recalculate" button: upserts `wellbeing_scores` with `onConflict: 'workspace_id,membership_id'`; creates `wellbeing_alerts` for <40 scores
- Team heatmap: color-coded grid per member
- Alert inbox: filter by severity, Resolve button sets `resolved_at` + `resolved_by`

### `src/components/enterprise/wellbeing/WellbeingScoreCard.tsx`
- Props: `{ workspaceId: string; userId: string }`
- Employee self-view: score donut visual, 5-component JSONB breakdown
- Leave recommendation hint shown when score < 60

## DB changes

New tables (migration `supabase/migrations/20260512210000_analytics_wellbeing_api_platform.sql`):
- `wellbeing_scores` — id, workspace_id, membership_id, score INT (0–100), components JSONB, calculated_at
- `wellbeing_alerts` — id, workspace_id, membership_id, alert_type, severity, message, triggered_at, resolved_at, resolved_by

RLS: members SELECT own row, admins ALL.
