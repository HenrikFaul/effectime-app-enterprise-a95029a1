## New edge function

### `supabase/functions/superadmin-hub/index.ts` (634 lines)
Auth: JWT ā†’ `user_roles` admin check on every request (403 if not admin).
Service role admin client used for all DB operations ā€” service role key never reaches client.

Actions implemented:
- `platform-overview`: workspace totals + user counts from auth schema + email queue stats
- `list-workspaces`: all enterprise workspaces + member counts + owner emails (up to 200)
- `workspace-action`: archive/unarchive/enable_recovery/disable_recovery/delete + audit event per action
- `list-feature-flags`: reads `platform_feature_flags` ordered by category/key
- `toggle-feature-flag`: updates `enabled` + `updated_by`/`updated_at`
- `list-cron-jobs`: reads `cron.job` (pg_cron schema), soft-fails if inaccessible
- `trigger-edge-function`: allowlist = ['sync-holidays','ms365-sync','send-scheduled-reports','cleanup-temp-users','cleanup-demo-workspace'] + audit event
- `locale-registry`: 5-locale static list + workspace counts + feature flag state
- `email-queue-status`: email_queue table count by status + last 10 entries
- `platform-version`: timestamp, supabase_url, function_count, runtime info

## DB changes (migration `20260512230000_platform_superadmin.sql`)

- `platform_feature_flags`: key UNIQUE, description, category, enabled BOOLEAN DEFAULT true, notes, updated_by, updated_at
- RLS enabled; 4 policies (SELECT/INSERT/UPDATE/DELETE) all guarded by `has_role(auth.uid(), 'admin')`
- `updated_at` trigger via `update_updated_at_column()`
- 10 seed rows: `locale.*` (4), `platform.*` (6)
