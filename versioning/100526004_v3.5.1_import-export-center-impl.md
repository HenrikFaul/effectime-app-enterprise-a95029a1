## Funkcionális deliverable

### 7 támogatott entitás

| Entity Key | Hungarian Label | Export | Import | Upsert | Egyedi kulcs |
|------------|-----------------|--------|--------|--------|--------------|
| `members` | Tagok | ✅ | ✅ | ✅ | email |
| `leave` | Szabadságok | ✅ | ✅ | ❌ | email + start_date + end_date + leave_type |
| `offices` | Telephelyek | ✅ | ✅ | ✅ | name |
| `work_categories` | Munkakategóriák | ✅ | ✅ | ✅ | name |
| `job_roles` | Munkakörök | ✅ | ✅ | ✅ | name + category_name |

### Export wizard funkciók

- Csoportosított mező-checklist (group field-enként)
- Kötelező mezők zárolva (lock ikon, nem kivehető)
- "Összes" / "Csak kötelező" preset gombok
- Formátum: XLSX (Excel XML) vagy CSV (UTF-8 + BOM)
- Import-kompatibilis sablon mód: asterisk required oszlopokon + guidance row
- Leave entitás: dátumtartomány + státusz szűrő (round-trip)
- Audit log entry: `export.created` action

### Import wizard funkciók (7 lépés)

1. **Útmutató** — instrukciók + üres sablon letöltés + jelenlegi adatok letöltés sablon formátumban + drag-drop upload
2. **Feltöltés** — fájl-info preview (név, méret, oszlop+sor szám)
3. **Oszlop-leképezés** — táblázat: fájl fejlécek vs system mezők, dropdown-nal módosítható
4. **Validáció Preview** — row-level státusz (✅/🟡/🔴), hibás cellák tooltipben, hibalista letöltés CSV-ként
5. **Importálási mód** — Csak új sorok / Frissítés is (upsert, opt-in)
6. **Megerősítés** — összegzés kártya + "Megértem" checkbox
7. **Eredmény** — sikeres/hibás darab, hibalista, hibák letöltése

### Validációk implementálva

- `REQUIRED_EMPTY` — kötelező mező üres
- `INVALID_EMAIL` — RFC 5322 ellenőrzés
- `INVALID_DATE` — YYYY-MM-DD, DD.MM.YYYY, YYYY/MM/DD elfogadott
- `INVALID_BOOLEAN` — true/false, igen/nem, 1/0, yes/no elfogadott
- `INVALID_NUMBER` — Number() parse
- `INVALID_ENUM` — machine value vagy display label match
- `BUSINESS_RULE` — pl. leave end_date < start_date
- `DUPLICATE_IN_FILE` — egyedi kulcs alapján két sor a fájlban
- `REFERENCE_NOT_FOUND` — pl. office_name nincs a workspace-ben (Edge Function)

### Auto column mapping

A validátor három fokozatú matching-et alkalmaz:
1. Exact key match (pl. `email` ↔ `email`)
2. Alias match (pl. `Email`, `e-mail`, `mail` ↔ `email`)
3. Label match (pl. "Teljes név" ↔ `display_name`)

Ha nincs talált, `__ignore__` az alapérték — a felhasználó manuálisan beállíthatja vagy kihagyhatja.

### Edge Function biztonsági ellenőrzések

- JWT verify (verify_jwt = true)
- `has_enterprise_role(workspace_id, user_id, ['owner', 'resourceAssistant'])` RPC ellenőrzés
- Workspace scoping minden insert/update-en
- Reference resolution csak workspace-en belül
- Audit események: `import.started`, `import.completed`, `import.completed_with_errors`

1. **Frontend**: standard Vite build + deploy
2. **Edge Function**: `supabase functions deploy import-entity-data`
3. **Database**: nincs új migráció szükséges (létező táblákat használunk)
4. **Smoke test**:
   - Settings → "Adatkezelés — Import / Export" → Export → Tagok → mezők kiválasztása → Letöltés → CSV-be megnyitás
   - Import → Tagok → Üres sablon letöltés → 1-2 sor hozzáadása → Feltöltés → 7-lépéses wizard végigjátszás → Eredmény check
