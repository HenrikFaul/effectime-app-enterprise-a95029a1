## Jira ticketek ebben a fázisban
- **SYN-148** — Leave types, holidays and work pattern configuration
- **SYN-147** — Conflict engine and validation messages for request submission

## Összefoglaló
Enterprise B2B modul 3. fázis: Konfiguráció kezelés és ütközésvizsgálat.

### Implementált funkciók

1. **Enterprise configuration DB schema** (SYN-148):
   - `enterprise_leave_types`: egyedi szabadság típusok (név, szín, fizetett, jóváhagyás szükséges, aktív, sorrend)
   - `enterprise_holidays`: munkaszüneti napok (dátum, név, évente ismétlődő)
   - `enterprise_blocked_dates`: tiltott napok (dátum, indoklás)
   - `enterprise_daily_rules`: napi szabályok (adott dátumra vagy hét napjára, max távollévő, min lefedettség, csapat szűrő)

2. **Configuration management UI** (SYN-148):
   - `LeaveTypeManager`: CRUD szabadság típusokra, szín választó, fizetett/jóváhagyás switch, aktiválás/deaktiválás
   - `HolidayManager`: ünnepnapok hozzáadása/törlése dátumválasztóval, évente ismétlődő opció
   - `BlockedDateManager`: tiltott napok hozzáadása/törlése indoklással
   - `DailyRuleManager`: napi szabályok (hét napja vagy adott dátum), max-off és min-coverage

3. **Conflict engine** (SYN-147):
   - `src/lib/conflictEngine.ts`: párhuzamos lekérdezés ünnepnapokra, tiltott napokra, napi szabályokra és meglévő kérelmekre
   - Ütközés típusok: `BLOCKED_DATE` (blocking), `HOLIDAY_OVERLAP` (warning), `MAX_OFF_EXCEEDED` (blocking), `MAX_OFF_WARNING` (warning), `SELF_OVERLAP` (warning)
   - Machine-readable kódok + human-readable magyar üzenetek

4. **LeaveRequestDialog integráció**:
   - Kétlépcsős submit: először "Ellenőrzés" (validáció futtatás), majd "Kérelem beküldése"
   - Ütközések vizuális megjelenítése: piros (blocking) vs sárga (warning)
   - Blocking ütközés esetén beküldés letiltva

5. **WorkspaceDashboard bővítés**:
   - Új "Szabályok" tab (ShieldAlert ikon, csak admin)
   - A tab tartalmazza: LeaveTypeManager, HolidayManager, BlockedDateManager, DailyRuleManager
