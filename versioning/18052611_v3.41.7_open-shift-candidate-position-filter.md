# v3.41.7 — Fix: Open Shift Candidate List Filtered by Position

**Date:** 2026-05-18
**Branch:** claude/analyze-repo-structure-dE8F4
**Type:** bugfix

## Root Cause

`useShiftCandidates` fetched all active workspace members and only used `businessRole`
as a scoring hint in the eligibility engine (±50 / −40 pts), never as a hard filter.
When "Optometrista" was selected, the result set included every active member ranked
by availability — so Erika Balogh (Senior Backend Developer) and Csilla Nagy
(Operations Lead) appeared in the top-3 with WRONG_ROLE warnings.

Simultaneously, `enterprise_member_role_allocations.business_role` was never consulted
during candidate lookup. Henrietta Fekete's position ("Optometrista") was only recorded
in that table — her `enterprise_memberships.business_role` was either null or
a different value — so she was never promoted to the top-3 even though she is one
of the two confirmed optometrists in the workspace.

The capacity view correctly showed 2 Optometrista headcount (Henrietta + Henrik) because
it reads from `enterprise_member_role_allocations`. The candidate list read only from
`enterprise_memberships.business_role` and applied no role-based exclusion.

## Fix

`src/hooks/useOpenShifts.ts` — `useShiftCandidates`:

1. **Added parallel query** to `enterprise_member_role_allocations` when `businessRole`
   is non-null. Fetches `membership_id` for all allocations matching the requested role.

2. **Hard-filter applied** after loading all memberships: when a role is specified, only
   members whose `enterprise_memberships.business_role` matches OR whose `membership_id`
   appears in the allocations result are kept. Members with unrelated positions are
   excluded entirely — they no longer pollute the candidate list.

3. **MemberInput.business_role normalisation**: for members matched exclusively via
   allocations (their memberships row has a null or different `business_role`), the field
   is set to the requested `businessRole` so the eligibility engine awards the +50
   role-match score bonus and does not emit a spurious `WRONG_ROLE` warning.

## Files Changed

- `src/hooks/useOpenShifts.ts` — `useShiftCandidates` rewritten to filter and
  score by position using both membership and allocation data sources

- When no role selected: all active members returned (unchanged behaviour)
- Both members matched via `enterprise_memberships.business_role` AND
  `enterprise_member_role_allocations` receive full +50 role-match score
