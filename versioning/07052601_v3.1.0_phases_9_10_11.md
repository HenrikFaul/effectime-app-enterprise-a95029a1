## Phase 9 — QA, Testing, and Release Safety Net

### 9.1 Test architecture

**Test runner**: Vitest v3.x (configured in `vitest.config.ts`); jsdom environment; `@testing-library/react` available for component tests when needed.

**Setup file**: `src/test/setup.ts` — mounts `@testing-library/jest-dom` matchers and provides a `window.matchMedia` stub.

**Included glob**: `src/**/*.{test,spec}.{ts,tsx}` — any file ending in `.test.ts(x)` or `.spec.ts(x)` under `src/` is automatically picked up.

**Alias support**: `@/` maps to `./src/` — the same alias used by the production build — so tests import production modules with the exact same resolution as the app.

### 9.5 Localization testing strategy

The `i18n.localization.test.ts` file runs on every `vitest run` invocation and enforces:

1. **Minimum key count floor** (100 keys) — catches accidental bundle truncation.
2. **5% gap tolerance** — allows active development without failing CI on small HU gaps, while flagging systematic missing coverage.
3. **Empty-EN + non-empty-HU** regression — catches cases where an EN key was blanked but its HU translation remains.
4. **Explicit critical-key assertions** — `common.save`, `header.help`, `header.language`, `organization.title`, `organization.tabs.structure`, `organization.tabs.chart` must always be present and non-empty in both locales.

**Adding a new language**: When a third locale (e.g., German `de`) is added, extend `i18n.localization.test.ts` to import the `de` bundle and repeat the parity assertions. The test file is the canonical localization quality gate.

## Phase 11 — Final Implementation Roadmap

### 11.1 Governing principles

1. **High-impact, low-risk first** — ship things that unblock others without touching risky shared state.
2. **Backend-first** — migrations and RLS policies before UI so the data layer is ready when the UI ships.
3. **Localization in lockstep** — every new string ships with both EN and HU keys; the 5% gap test enforces this.
4. **QA gate at every wave** — `vitest run` must pass before merging each wave.
5. **Feature flags as circuit breakers** — every new capability defaults to off in production.
