## 1. OpenShiftManager — targeted notification toast fix

### Bug
Both the "Kiválasztottak értesítése" and "Meghirdetés az összes megfelelőnek" buttons
called `handlePost(true/false)` but both showed the same
`t('open_shifts.posted_success')` toast ("broadcast sent to all"). The distinction
was invisible to the manager.

### Fix
- `handlePost` now branches on `targetOnly`:
  - `targetOnly = true` → `toast.success(t('open_shifts.notify_selected_success', { count }))` — shows count of notified candidates
  - `targetOnly = false` → existing broadcast toast (unchanged)

### Visual distinction
- "Kiválasztottak értesítése" button changed from `variant="secondary"` to `bg-violet-600 hover:bg-violet-700 text-white` — clearly distinct from the primary broadcast button.

## 2. EmployeeMonthView — two-mode gated edit system

### Problem
- Employees could change their availability by clicking days even without entering edit mode — a silent, accidental-edit risk.
- There was only one edit mode; worktime editing and availability editing were mixed.

### Solution

**State change:**
```typescript
// Before
const [editMode, setEditMode] = useState(false);

// After
type EditMode = 'none' | 'worktime' | 'availability';
const [editMode, setEditMode] = useState<EditMode>('none');
const [availPaintStatus, setAvailPaintStatus] = useState<AvailabilityStatus>('available');
```

**Edit dropdown:** The single "Szerkesztés" button is now a DropdownMenu offering two options:
- *Munkaidő szerkesztése* → `setEditMode('worktime')` — existing segment/schedule editing
- *Elérhetőség szerkesztése* → `setEditMode('availability')` — new paint mode

**Availability paint mode:**
- Violet toolbar shows 3 status buttons (Elérhető / Preferált / Nem elérhető); user selects active paint color.
- Click or drag paints day(s) with the selected status via `useUpsertAvailability`.
- Past dates (< today) are blocked: cursor shows `not-allowed`, `handleAvailPaint` returns early without writing.
- "Kész" (Done) button exits the mode.

**Removed:** `canToggleAvail` free-click logic, `STATUS_CYCLE`, `nextAvailStatus`, `handleToggleAvailability` — all replaced by the gated paint mode.
