## Why scaffolds first

A complete professional translation of ~4000 keys × 3 locales is a
content task requiring native-speaker review (HR-domain phrasing,
jurisdiction-specific legal terminology). Shipping scaffolds with
English fallback:

1. Lets the UI render in the selected locale today.
2. Lets the platform admin localize key strings incrementally as
   translation budget allows.
3. Doesn't gate new features on "do we have a German translator?".

Each scaffold file is byte-for-byte the contents of `en.ts` with the
const name renamed. The runtime lookup chain returns the source
locale's value for any key; missing keys fall back to English.
