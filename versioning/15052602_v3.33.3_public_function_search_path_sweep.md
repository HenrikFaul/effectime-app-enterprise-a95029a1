## Why now

The v3.33.2 hotfix patched the 4 new functions I authored. The 10
above were pre-existing but flagged by the same advisor lint. The
audit cycle would have called this out at the next pass anyway —
batching it now keeps the advisor clean for `public` schema and
prevents the warning list from masking future regressions.

## Risk

Near-zero. `SET search_path` on a function body that uses qualified
names changes nothing at runtime — it just hardens resolution
against a malicious session-level search_path. The functions were
read in full before edit; none rely on extension functions or
unqualified non-public schema objects.
