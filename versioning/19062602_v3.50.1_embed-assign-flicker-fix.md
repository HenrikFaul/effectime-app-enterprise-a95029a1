# v3.50.1 — Embed: kill the assign-panel flicker (optimistic writes)

**Date:** 2026-06-19
**Branch:** claude/embed-assign-flicker-fix
**Type:** Bugfix (frontend UX / rendering)

## Goal
Remove the flicker the customer reported in the embed capacity "daily scheduling" side-panel: assigning a person flashed the loading skeleton, the cell counts strobed (1‑2‑3), the whole panel locked, and only after a full reload was it usable again.

## Root causes
1. Every write (`handleAssign` / `handleRemove` / `handleSmartSuggest`) called `load()`, which set `loading=true` → the grid swapped to `<LoadingSkeleton/>` (4 staggered-opacity bars — the visible "vibrate").
2. No optimistic update — the count only changed once the refetch resolved, so values jumped around (worse when racing multiple quick clicks).
3. A single global `saving` boolean disabled **every** button in the panel for the duration of each RPC.
4. `WriteSheet` was declared inside the component body and rendered as `<WriteSheet/>`. A new function identity each render = a new component type, so React **unmounted+remounted** the Radix `Sheet` on every data change (re-triggering the slide-in animation).

## Fix (`src/components/embed/EmbedCapacityView.tsx`)
- **Optimistic local state**: assign/remove/suggest mutate `data.shift_assignments` immediately (assign mirrors the server upsert: drop any existing same-day shift for that user, then add). Counts, slots, and the grid update instantly.
- **Silent refresh**: `load({ silent?: boolean })` — post-write reloads pass `{ silent: true }` and never toggle `loading`, so no skeleton swap; the grid/panel reconcile in place against server truth. Initial load + week/month navigation still show the skeleton.
- **Per-key in-flight tracking**: replaced `saving: boolean` with `savingKeys: Set<string>` keyed by `user_id` (assign), shift `id` (remove), or `suggest:<rule>:<date>`. Only the clicked row spins; the rest of the panel stays interactive, so several people can be assigned in quick succession.
- **No remount**: the panel is now rendered with `{WriteSheet()}` (a plain inline call — it uses no hooks) instead of `<WriteSheet/>`, so the Sheet subtree reconciles across renders instead of remounting.
