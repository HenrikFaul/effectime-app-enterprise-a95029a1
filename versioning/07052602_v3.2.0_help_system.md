# 07052602 — v3.2.0 Self-Updating Context-Aware Help System

Date: 2026-05-07

## Goal
Build a help system that regenerates itself from the GitHub repository on every release, exposes context-aware articles in a redesigned right-side drawer, supports drag-from-? element targeting, full-text search with autocomplete, and EN/HU localization.

## Components

### Database (`supabase/migrations/...help_system...sql`)
- `help_releases` — one row per regeneration run (version_tag UNIQUE, commit_sha, status enum, summary, error, triggered_by, started_at/completed_at).
- `help_articles` — stores generated content per (topic_key, locale).
  - Columns: title, summary, body_md, route, anchor_id, taxonomy, tags[], synonyms[], related_topics[], release_id FK, source_release_tag, content_hash, is_system_generated, search_tokens (tsvector), last_generated_at.
  - Indexes: (anchor_id, locale), (route, locale), GIN on tags, GIN on search_tokens.
  - Trigger `help_articles_search_trigger` rebuilds tsvector on insert/update with weighted A/B/C across title, summary+tags+synonyms, body.
- RLS: public SELECT, no INSERT/UPDATE/DELETE policy (service role only).

### Edge function `supabase/functions/help-regenerator/index.ts`
- HMAC SHA-256 verifies `x-hub-signature-256` against `GITHUB_RELEASE_WEBHOOK_SECRET` (skipped if secret unset).
- Maps GitHub `release` event payload to `{ repo, ref, version_tag, commit_sha }`. Manual POST also accepted.
- Fetches `CHANGELOG.md` and last 10 `versioning/*.md` via raw.githubusercontent.com.
- Calls Gemini 2.0 Flash (`generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent`) with:
  - System prompt instructing user-facing tone, taxonomy enum, EN+HU variants, structured JSON array.
  - `responseMimeType: application/json`, temperature 0.3.
- Upserts each variant on conflict `(topic_key, locale)`; computes SHA-256 content hash.
- Updates the `help_releases` row to `succeeded`/`failed` with summary + error.

### Frontend
- `src/lib/help/registry.tsx` — adds `selectedAnchorId` (drag-target wins over current page anchor) and `dragging` state to context.
- `src/components/help/HelpButton.tsx` — `pointerdown` → tracks movement; > 6px = drag mode. While dragging, `elementFromPoint` finds the nearest `[data-help-region]` and toggles `.help-target-hover`. On `pointerup` over an anchor, opens drawer with that anchor selected. Pure click opens page-level help.
- `src/components/help/HelpDrawer.tsx` — redesigned: glass background, search bar with X-clear, debounced live results (200 ms, dedup by topic_key, prefer active locale), crumbs row, markdown body via `react-markdown`, release tag footer.
- `src/lib/help/useHelpArticles.ts` — `useHelpArticleByAnchor(anchorId, locale)` with EN fallback; `useHelpSearch(query, locale)` ilike across title/summary/body.

### Styles (`src/index.css`)
- Existing `.help-highlight-ring` (page-anchor pulse) preserved.
- New `.help-target-hover` — primary-tinted ring + brightness boost while drag-hovering.
- Both honor `prefers-reduced-motion`.

### Localization
- Reuses existing `enterprise_translation_overrides` resolver in `I18nProvider`.
- DB articles are pre-localized (Gemini generates EN + HU together). Drawer queries active locale first, falls back to EN per row.
- No new locale tables; respected the existing `profiles.preferred_locale` flow.

## How to wire the GitHub webhook
1. Repo Settings → Webhooks → Add webhook
2. Payload URL: `https://oezlzzmzzvbvinuysxaz.supabase.co/functions/v1/help-regenerator`
3. Content type: `application/json`
4. Secret: value of `GITHUB_RELEASE_WEBHOOK_SECRET`
5. Events: "Releases" only.

Manual seed (any logged-in admin terminal):
```bash
curl -X POST https://oezlzzmzzvbvinuysxaz.supabase.co/functions/v1/help-regenerator \
  -H 'content-type: application/json' \
  -d '{"repo":"OWNER/REPO","ref":"main","version_tag":"v3.2.0"}'
```
