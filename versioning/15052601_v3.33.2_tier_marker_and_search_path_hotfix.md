# v3.33.2 — Hotfix: tier-change RPC marker + search_path on v3.33.1 functions

**Date:** 2026-05-15
**Branch:** `claude/review-project-requirements-lR0Wp`
**Scope:** 2 hotfixes for v3.33.1 regressions, found by continuing the audit cycle.

## What broke

### 1. `superadmin_change_workspace_tier` did not arm the immutability guard

In v3.33.1 we shipped:
- A trigger `enforce_tier_id_immutability` on `tenant_subscriptions` that
  blocks any `UPDATE` setting `tier_id` unless the session-local
  `app.tier_change_rpc_active = 'true'` is set.
- An expectation that `superadmin_change_workspace_tier` (the only
  legitimate writer) sets that marker inside its body.

Reading the live RPC body via `pg_get_functiondef` revealed the marker
was **missing**. Result: every tier change in production would have
failed with `tier_id is immutable; use public.superadmin_change_workspace_tier()`
— the very RPC the user was calling.

The bug never reached production because the audit caught it before
QA exercised the path. Live fix applied via MCP, plus disk migration
+ regression tests.

### 2. Four v3.33.1 functions lacked `SET search_path`

The Supabase security advisor (`function_search_path_mutable`) flagged:
- `enforce_tier_id_immutability`
- `validate_tier_feature_keys`
- `validate_feature_dependencies`
- `require_feature_id`

Without `SET search_path`, an attacker with influence over session
search_path could shadow `public.features` / `public.tenant_subscriptions`
during trigger execution. All 4 now declare `SET search_path TO 'public'`.

## Why the bugs happened

I added the trigger before checking the RPC body — the assumption
"the RPC sets the marker because we agreed it would" was unverified.
And I authored the new functions without copying the `SET search_path`
clause that other recent functions use.

## What ships

### DB migration (applied + on disk)
`supabase/migrations/20260515002644_v3_33_2_tier_marker_and_search_path_hotfix.sql`

- Replaces `superadmin_change_workspace_tier` with a version that calls
  `PERFORM set_config('app.tier_change_rpc_active', 'true', true)`
  immediately before its UPDATE/INSERT.
- Replaces all 4 trigger/helper functions to declare `SET search_path TO 'public'`.

### Regression-net tests
`src/test/migrationInvariants.test.ts` — 21 tests scanning
`supabase/migrations/` and asserting LATEST definitions hold:

- Strict tier_key contract (no DEFAULT literal, raises on NULL,
  raises on unknown, no `ORDER BY sort_order` fallback after the
  strict check, arms the tier-change guard).
- Tier-id immutability trigger references the marker + declares
  `SET search_path`.
- `superadmin_change_workspace_tier` actually calls
  `set_config('app.tier_change_rpc_active', 'true', true)` and
  writes a `platform_audit_events` row.
- Both delta-validation triggers use the `<> ALL(COALESCE(OLD.col, '{}'))`
  pattern (so existing typos don't block unrelated UPDATEs).
- All 6 v3.33.x-touched functions declare `SET search_path`.

These tests would have caught both v3.33.2-class bugs at PR time.

### Lessons
Two new entries appended to `codingLessonsLearnt.md`:
- `[LESSON-TRIGGER-PAIR-001]` — never ship an enforcement trigger
  without first verifying every legitimate writer arms its guard.
- `[LESSON-PG-SEARCH-PATH-001]` — every new SECURITY DEFINER /
  trigger function must declare `SET search_path` (advisor lint).
