## Summary

Extends the Business Intelligence prompt architecture (introduced in v3.35.0) with seven domain-specialist prompt files, one new template, and updates to the master controller and README routing tables. Phase 2 fills the remaining BI lifecycle gaps: predictive analytics, domain-specific deep analysis (financial, compliance, workforce), platform health, BI governance, and narrative craft.

---

## New files

### Specialist prompts (7 new)

**`prompts/predictive_signals.md`**
Leading indicator discovery protocol, Effectime leading indicator map (four prediction domains: disengagement/turnover, compliance failure, scheduling/coverage, platform churn), early warning system design (5-step), threshold-based alert structure (Green/Amber/Red), version-aware prediction caution, and predictive output format.

**`prompts/financial_analytics.md`**
Complete Effectime financial metric catalog (labor cost, payroll accuracy, marketplace economics — 15+ metrics), financial analysis protocol (5 steps), cost variance decomposition into volume/rate/efficiency drivers, payroll correction pattern analysis, marketplace fill-rate economics, and financial output format.

**`prompts/compliance_analytics.md`**
Effectime compliance metric catalog (10 metrics), compliance score decomposition protocol (4 steps: rule coverage, violation type breakdown, audit trail completeness, WTD-specific analysis), audit readiness 7-dimension scorecard, violation pattern analysis (4 tests), regulatory calendar awareness for EU operating markets (HU/DE/AT/CZ/SK/PL/RO), and compliance output format.

**`prompts/workforce_planning.md`**
Capacity analysis protocol (supply vs. demand framework, effective capacity formula, coverage ratio thresholds), headcount sufficiency modeling (FTE calculation from demand hours and effective hours per employee), turnover impact modeling, seasonal capacity planning, scheduling efficiency metrics and pattern analysis, workforce composition risk analysis (5 risk types), and workforce planning output format.

**`prompts/platform_health.md`**
Platform health metric catalog (workspace adoption, feature engagement, tier/commercial, integration health, churn risk — 20+ metrics), platform health assessment protocol (3-horizon framework: current state/recent trend/forward signal), at-risk workspace identification (5-signal model), integration health deep dive (transient vs. persistent failures), churn signal escalation protocol by tier, and platform health output format.

**`prompts/bi_governance.md`**
Metric definition consistency audit (8-item checklist), metric lifecycle states (Draft/Active/Deprecated/Retired/Contested), metric deprecation protocol (5 steps), definition change protocol (5 steps), BI output quality review (3-tier gate: factual accuracy / analytical rigor / governance compliance), BI standards registry (10 standards), metric conflict resolution protocol (6-step), and governance output format.

**`prompts/narrative_storytelling.md`**
The BI story arc (5-part: Situation/Complication/Question/Answer/Evidence), audience-specific narrative adaptation (board/executive/HR leadership/workspace manager), evidence selection hierarchy (definitive/supporting/contextual/tangential), uncertainty disclosure protocol, 7 BI narrative anti-patterns with explanations (data dump, chart tour, balanced both-sides, hedge sentence, buried lead, passive voice shield), narrative length standards by output type, and narrative output format.

### Templates (1 new)

**`templates/board_report_template.md`**
9-section formal board report structure: period summary, strategic KPI scorecard, performance highlights, risk register, operational performance detail (5 domains), forward guidance (leading indicators + expected outcomes + board-level decisions), governance and data integrity notes, metric definitions appendix, period data archive reference.

## Design notes

Phase 2 follows the same design principles as Phase 1 (v3.35.0): every file is repository-aware, version-aware, Effectime-specific, and anti-hallucination disciplined. The seven new domain specialists share the same 3-block output contract (BI Context + Analysis + Confidence Assessment) established in SYSTEM.md.

The compliance_analytics.md file includes EU-jurisdiction-specific regulatory context (WTD, operating markets) because Effectime's primary customer base is EU-based (HU, DE, AT, CZ, SK, PL, RO) and compliance failures in these markets have legal consequence.

The bi_governance.md file introduces the metric lifecycle state model (Draft/Active/Deprecated/Retired/Contested) which is not yet enforced by application code but is a governance standard for all BI outputs produced under this system.
