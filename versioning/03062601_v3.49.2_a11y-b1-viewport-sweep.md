# v3.49.2 — A11y B1: viewport unit sweep (`h-screen` → `h-dvh`)

**Date:** 2026-06-03
**Lens:** Senior Accessibility Specialist + Senior UI/UX Designer
**Batch:** B1 from the v3.49.1 a11y roadmap.

## What
Mechanical replace across 16 files:
- `h-screen` → `h-dvh`
- `min-h-screen` → `min-h-dvh`
- `max-h-screen` → `max-h-dvh`

## Why
WCAG 2.2 SC 1.4.10 (Reflow). On mobile Safari/Chrome the address-bar overlay
makes `100vh` taller than the visible viewport, clipping primary content
behind browser chrome. `100dvh` (dynamic viewport height) tracks the visible
area correctly and is identical to `100vh` on desktop — zero desktop regression.

## Files (16)
App shell, toast, workspace dashboard, and every full-height page route
(Admin, Auth, CandidateBook, EmbedPage, Enterprise, Landing, NotFound,
Profile, Reseller, ResetPassword, Superadmin, Unsubscribe).

## Anti-regression
- No logic touched.
- Tailwind `*-dvh` is supported natively in the project's Tailwind version.
- Desktop layouts unchanged (dvh = vh without dynamic chrome).

## Follow-ups
B2–B5 icon-button `aria-label` batches, B6 id sweep, B7 EmptyState primitive,
C-phase UX copy audit.
