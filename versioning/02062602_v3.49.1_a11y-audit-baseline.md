# v3.49.1 — WCAG 2.2 accessibility audit baseline (B-phase)

**Date:** 2026-06-02
**Lens:** Senior Accessibility Specialist + Senior UI/UX Designer

## Why audit-only
Per the no-regression mandate, fixing 109 icon-button labels + 26 viewport units + 39 hardcoded IDs in one PR is too broad to QA. This PR captures the baseline; B1–B7 batches in later PRs execute the fixes.

## Deliverable
`db-audit/a11y_audit_v3.49.1.md` — severity matrix, WCAG SC mapping, 7-batch fix roadmap, verification protocol.

## Top findings
- **P0 Critical (109)** — icon-only `<Button size="icon">` without `aria-label` (60 files)
- **P1 High (26)** — `h-screen` instead of `h-dvh` (16 files, mobile viewport bug)
- **P2 Medium (39)** — hardcoded `id="..."` in 10 list/dialog files
- **P3 Medium** — empty-state inconsistency (no `<EmptyState>` primitive)

## Anti-regression
Zero code changed. Documentation-only.

## Follow-ups
- v3.49.2: B1 viewport sweep (`h-screen` → `h-dvh`)
- v3.49.3+: B2–B5 icon-label batches
- v3.50.x: B6 ID sweep, B7 EmptyState primitive
- v3.49.x: C-phase UX copy audit (still TODO from D-scope request)
