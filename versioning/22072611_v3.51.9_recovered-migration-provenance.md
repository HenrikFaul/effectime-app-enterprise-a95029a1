## Problem and evidence

The local migration chain omitted two DDL files that are present in the linked
history. Generated types expose their tables/functions and a later local index
migration references `plugin_webhook_events` unconditionally, so clean replay
cannot reach the current schema without the original sources. Read-only history
retrieval recovered the exact remote statements for migration IDs
`20260514124827` and `20260514194031`.

## Implemented boundary

- Restores both historical sources at their original version/name paths without
  changing a byte of SQL.
- Freezes raw byte length and SHA-256 in an exact two-entry manifest. The
  dependency-free verifier rejects unknown fields, missing/reordered entries,
  traversal, path/name disagreement, symlinks, non-files, length drift and hash
  drift.
- Adds file-specific LF attributes so Git checkout conversion cannot invalidate
  or silently change the raw-byte attestation across Windows and Linux.
- Intentionally updates the exact generated-schema debt baseline only for the
  five tables and five functions proven by these sources.
- Adds the unit and current-tree byte gate to the mandatory GitHub Quality Gate.
