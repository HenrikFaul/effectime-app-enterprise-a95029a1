# v3.38.0 — Generic BI Framework: Portable Drop-In Prompt Architecture

**Date:** 2026-05-16
**Version:** 3.38.0
**Type:** Tooling addition — no application code changed

---

## Summary

Creates `BI_FRAMEWORK/` — a completely self-contained, project-agnostic Business Intelligence prompt architecture that can be dropped into any repository and used immediately without modification. All 29 files are generic: no product names, no domain-specific table references, no technology assumptions. Project teams configure it via `SETUP.md` and the fillable Project Context block in `SYSTEM.md`.

This is a parallel addition alongside `business_intelligence/` (Effectime-specific). Neither folder touches the other.

## Key Design Decisions

- **All placeholders bracketed** — every project-specific value uses `[bracket notation]` to signal configuration required.
- **SETUP.md is new** — the Effectime version assumed inline knowledge; the generic version needs explicit onboarding.
- **Seasonal pattern library is a framework, not a catalog** — the Effectime version had 16 documented patterns; the generic version provides the framework and domain-specific starting prompts to investigate.
- **churn_risk_scoring.md has privacy section before scoring protocol** — Section 2 on privacy appears before Section 1 on the scoring steps, ensuring privacy constraints are read before any scoring output is produced.
- **Model-agnostic** — all files are plain Markdown; no API dependencies, no Claude-specific syntax.
