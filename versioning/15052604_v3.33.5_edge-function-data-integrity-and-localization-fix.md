# v3.33.5 — Edge-function data integrity, TOCTOU hardening, and localization fixes

**Date:** 2026-05-15
**Branch:** claude/architect-team-session-UWfpK
**Type:** Bug fix (no new features)

---

## Goals

Close seven correctness and security gaps discovered during the full-system codebase audit
(phases 3–8 of the multi-phase audit initiated in this session):

1. **GDPR export wrong column** — `security-admin` queried `enterprise_attendance_periods`
   and `wellbeing_scores` by `user_id`, but both tables use `membership_id`.
2. **Wellbeing hook schema mismatch** — `useWellbeing.ts` selected `period_start`,
   `period_end` (not in `wellbeing_scores`) and `metadata` (not in `wellbeing_alerts`).
3. **Payroll lock trust boundary + TOCTOU** — `lock-period` action accepted `userId`
   from the request body instead of the authenticated JWT, and the check-then-update
   pattern was not atomic.
4. **Silent markRun failure** — `send-scheduled-reports` `markRun()` did not check
   `.error` on the Supabase update, leaving schedule status silently unsynchronized.
5. **Audit INSERT after workspace DELETE** — `superadmin-hub` workspace-delete action
   wrote the audit row after the cascade delete, which could silently fail if the FK
   had already been cleaned up.
6. **allocationsRes unchecked** — `create-instant-enterprise-member` ignored errors
   from the `enterprise_member_role_allocations` Promise.all query.
7. **Hardcoded Hungarian strings in LeaveCalendar** — `WEEKDAY_LABELS`, `'Ismeretlen'`,
   and `'Minden csapat'` violated the localization mandate; all 8 locales updated.

## Implementation details

### Fix 1 — GDPR export schema mismatch (security-admin)

`enterprise_attendance_periods` and `wellbeing_scores` both use `membership_id` as their
user link, not `user_id`. The GDPR subject-access export was silently returning empty
arrays for attendance and wellbeing data for any user. Fixed by querying with
`targetMembership.id` (the membership row is already fetched for the workspace-guard
check at the top of the handler).

### Fix 2 — Wellbeing hook schema mismatch

`wellbeing_scores` table schema (migration `20260512210000`) has no `period_start` or
`period_end` columns. `wellbeing_alerts` has no `metadata` column — it has `message`
instead. The queries would succeed (PostgREST ignores unknown select columns) but the
interface properties would be `undefined` at runtime, causing subtle rendering bugs.

### Fix 3 — Payroll lock trust boundary + TOCTOU

Two issues in `lock-period`:
- `locked_by` was set to `body.userId` which the caller supplied — any authenticated
  admin could claim to lock on behalf of another user. Fixed by using `user.id` from
  the verified JWT.
- The fetch-then-update pattern had a race window. Added `.eq('status', 'open')` to
  the UPDATE WHERE clause and check that `lockedRows.length > 0`. If a concurrent
  request locks the same period first, the UPDATE touches 0 rows and we return 409.

### Fix 4 — markRun silent failure

`markRun()` used bare `await` — Supabase JS never throws on DB errors. Destructured
`{ error: updateErr }` and added a `console.error` on failure so ops can observe
when schedule status tracking breaks.

### Fix 5 — Audit INSERT before workspace DELETE

On `workspace-action: delete`, the audit row was inserted after the DELETE. If the
workspace cascade already removed the row that the FK constraint on
`enterprise_audit_events.workspace_id` pointed to, the insert would fail silently.
Moved the audit INSERT before the DELETE so the FK target still exists.

### Fix 6 — allocationsRes error swallowed

The fourth Promise.all result was destructured but `allocationsRes.error` was never
checked. Added a guard that returns 500 if the query failed, consistent with the
other three checks in the same block.

### Fix 7 — LeaveCalendar localization

- Removed module-level `const WEEKDAY_LABELS` (outside the component, inaccessible
  to `t()`).
- Added `weekdayLabels` as a `useMemo` inside the component using 7 new i18n keys.
- Replaced two `'Ismeretlen'` fallbacks with `t('leave_calendar.unknown')`.
- Replaced `'Minden csapat'` SelectItem text with `{t('leave_calendar.all_teams')}`.
- All 9 new keys added to all 8 locale files in this commit.
