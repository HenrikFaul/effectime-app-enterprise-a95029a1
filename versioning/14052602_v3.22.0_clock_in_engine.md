## Why this matters

Replaces physical time-clock hardware (Dormakaba / HID Global /
Suprema) which costs €500–€3,000 per device. Smartphone-based
verification with smartphone-native methods (GPS geofence, NFC tap,
rotating QR). Per the strategy doc: $4.2B hardware-replacement market.

EU regulatory context: ECJ C-55/18 ("Deutsche Bank") requires employers
to track all working time. This is a regulatory compliance driver, not
just convenience.

## Architecture

| Method | Validation | Spoof-resistance |
|---|---|---|
| **GPS** | Server-side haversine distance vs geofence center | Server-only; client lat/lng cannot bypass |
| **QR** | Rotating 60-second code from `qr_clock_sessions` | Photo-sharing fraud limited by 60s TTL |
| **NFC** | Matches `enterprise_offices.clock_in_nfc_tag` | Physical proximity required |
| **Manual** | Always unverified, flagged for manager review | Backstop for malfunctions |

## DB structure

| Object | Purpose |
|---|---|
| `clock_events` | Append-only attendance log |
| `qr_clock_sessions` | Rotating QR codes per office, 60s TTL |
| `enterprise_offices.geofence_lat/lng/radius_m/clock_in_nfc_tag` | Site configuration |
| `haversine_km(lat1, lon1, lat2, lon2)` | IMMUTABLE distance helper |
| `clock_generate_qr(office_id, ttl_seconds)` | Manager-only QR generator |
| `clock_event(...)` | Per-method validating attendance recorder |

`clock_events` table is RLS-scoped (member sees own; owner/RA sees team;
admin sees all) and direct-INSERT-blocked. The RPC is the sole writer.
