## What's new

`wellbeing_calculate_scores(workspace_id)` SECURITY DEFINER RPC. Manager
(owner / resourceAssistant) or platform admin can run it on demand. For
each active member it computes:

| Component | Weight | Data source |
|---|---:|---|
| Overtime ratio | 30% | `enterprise_attendance_segments` last 90 days, `segment_type='overtime'` vs total |
| Weekend density | 25% | Sat/Sun segments vs total (ISO DOW 6,7) |
| Leave utilization | 20% | `leave_requests.status='approved'` days vs `enterprise_leave_quotas.initial_balance` |
| Schedule stability | 15% | Neutral 70pts placeholder (v3.23.x: wire to schedule-change log) |
| Recovery score | 10% | Neutral 70pts placeholder (v3.23.x: cross-shift gap analysis) |

Score is clamped 0–100. Inserts `wellbeing_scores` row + fires:
- `low_wellbeing_score` alert (severity high) when score < 40.
- `overtime_warning` alert (severity medium) when score 40–60 and overtime ratio > 20%.

Weight overrides per workspace via `enterprise_workspaces.wellbeing_weights jsonb`.
