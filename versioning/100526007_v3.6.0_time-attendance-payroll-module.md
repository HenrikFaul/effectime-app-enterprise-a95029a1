## Mit szállítottunk

Egy teljes körű **Időnyilvántartás + bérelőkészítés** modult, ami azonnal használható manuális idő-rögzítésre, és architekturálisan készen áll a jövőbeli kártyás/badges-es automatikus check-in / check-out integrációra.

### A felhasználói folyamat egy mondatban
- **Tag**: napi szegmensek rögzítése (több is naponta), túlóra/éjszaka/hétvége flag, készenlét + behívás szétválasztva, havi benyújtás
- **Admin**: csapat-szintű áttekintő, jóváhagyás/visszaküldés/zárolás, bérszámfejtésnek strukturált XLSX/CSV export
- **Bérelőkészítés**: stabil 23 oszlopos exportfájl, audit-batch-ekkel, csak zárolt időszakok továbbíthatók

## Üzleti szabályok megvalósítva

| Szabály | Hol valósul meg |
|---------|------------------|
| Több szegmens / nap | `enterprise_attendance_segments` + DayEditorDialog |
| Túlóra szétválasztva | `segment_type='overtime'`, külön `overtime_hours` és `weekend_overtime_hours` |
| Hétvégi munka | `is_weekend` flag automatikusan beállítva Sat/Sun-ra |
| Éjszakai munka | `is_night` flag, opcionális automatikus 22:00–06:00 detektálás (`nightHoursInRange`) |
| Készenlét × 0.20 | `attendance_recompute_totals` SQL függvény |
| Behívás külön | `segment_type='oncall_intervention'`, levonódik a standby raw-ból |
| Havi konfirmáció | `transitionPeriod(... 'submitted')` + state machine |
| Szabadság-tudatos elvárt | `calc_leave_days(...)` × 8 → `expected_after_leave` |
| Audit | `enterprise_attendance_audit` minden mutáció után |
| Bérexport | `attendance_payroll_export` RPC + XLSX/CSV download |

## Új coding lessons

- `[LESSON-RPC-090]` — Komplex számítómotorok PostgreSQL `SECURITY DEFINER` függvénnyel: a számítás a server-en hiteles, kliens-oldali preview logika 1:1 tükrözése biztosítja a UX gyorsaságát anélkül hogy elromolhatna az export
- `[LESSON-AUDIT-091]` — Append-only audit table policy: csak SELECT policy, nincs INSERT/UPDATE/DELETE policy — minden írás SECURITY DEFINER RPC-n keresztül történik, így UI nem tudja kicserélni a múltat
- `[LESSON-DUAL-SOURCE-092]` — Forward-compatible field-ek: `segments.source ('manual'/'device')` + `segments.device_event_id` mostantól bekerülnek, így a jövőbeli automatikus integráció nem igényel séma-átalakítást, csak új ingest table + edge function-t

## Smoke test forgatókönyv

1. Workspace → top nav → **Időnyilvántartás**
2. Tag nézet: nyiss meg egy napot → adj hozzá egy szegmenst (09:00–17:00, normál) → mentés → KPI card frissül
3. Adj hozzá egy túlóra szegmenst egy szombatra (`is_weekend` automatikusan be) → "Hétvégi túlóra" KPI nő
4. **Készenlét rögzítése** → szombat 08–20 → KPI: 12h raw, 2.4h compensated
5. Vissza a napra: adj egy `oncall_intervention` szegmenst 10:00–12:00 → KPI: standby 10h, intervention 2h, compensated 2h
6. **Benyújtás** → státusz `submitted`
7. Admin: tab váltás "Csapat áttekintés" → soron **Jóváhagy** → státusz `approved` → **Zárol** → `locked`
8. **Bérelőkészítés export (XLSX)** → letöltődik a fájl, státusz `exported`
9. Excelben megnyitva: 23 oszlop, számok stimmelnek a KPI-vel
10. Új hónap-választás → új period létrehozódik draft-ban, ciklus újraindul
