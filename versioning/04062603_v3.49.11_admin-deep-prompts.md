# v3.49.11 — Admin & Superadmin deep technical prompts

**Date:** 2026-06-04
**Lens:** Systems analyst + documentation engineer.

## Why
The v3.49.10 audit surfaced that **only one** admin surface (Feature/Tier management) had a deep technical prompt (`docs/tiering/superadmin_spec.md`). The other 7 admin surfaces had no per-feature regression-proof specification — meaning any AI dev session touching them was working from source code inference alone. That is the exact pattern that produced the v3.21.2 admin-leak regression.

## What shipped
- `AI_PROMPTING_FOLDERSTRUCTURE/admin/` — new folder, 9 files:
  - `00_INDEX.md` — router, read order, surface map, edge-function action table, authoring rules
  - `01_workspace_admin_dashboard.md` — `/admin` Dashboard tab
  - `02_workspace_admin_users.md` — `/admin` Users tab (list / role toggle / delete)
  - `03_superadmin_control_plane_shell.md` — `/superadmin` shell + ops panels
  - `04_superadmin_tenant_management.md` — workspaces tab (list / archive / delete / tier change)
  - `05_superadmin_impersonation.md` — recovery mode (security-critical)
  - `06_superadmin_feature_tiers_tab.md` — supersedes `docs/tiering/superadmin_spec.md`
  - `07_superadmin_platform_audit_log.md` — audit viewer with immutability invariants
  - `08_admin_password_policy_and_reset.md` — shared policy for workspace + platform resets
- `AGENTS.md` — read order extended with the new INDEX (step 5).
- `docs/tiering/superadmin_spec.md` — marked SUPERSEDED with a redirect banner; content preserved for wireframe history.

## Anti-regression
- No source code touched.
- No DB migration.
- Existing `docs/tiering/superadmin_spec.md` content kept verbatim under the banner.
