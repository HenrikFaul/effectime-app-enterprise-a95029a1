# v3.12.0 — SOC 2 / ISO 27001 Technical Controls — Security Center

**Date:** 2026-05-12
**Branch:** claude/create-software-documentation-O7kj1
**Type:** New feature

## Scope

Enterprise security & compliance module implementing key SOC 2 Type II / ISO 27001 technical controls: enhanced immutable audit log, configurable data retention policies, GDPR data subject rights tooling, and session management.

## Components delivered

### `src/components/enterprise/security/SecurityCenter.tsx`
- Props: `{ workspaceId: string; userId: string; isAdmin: boolean }`
- 3-tab structure: Audit Log | Data Retention | GDPR

**Audit Log tab:**
- Reads `enterprise_audit_events` (now with `ip_address` + `user_agent` columns)
- Filters: action type (8 categories), date range, actor search
- Expandable rows: prev_state / new_state JSON diff
- Paginated (50/load), "Load more" button
- CSV export of filtered results

**Data Retention tab:**
- CRUD for `data_retention_policies` per table
- Tables: audit_events, api_usage_logs, wellbeing_scores, etc.
- `is_active` defaults to false — no automatic deletion until explicitly activated post legal review
- Info callout warning about legal review requirement

**GDPR tab (Art. 17 + Art. 20):**
- Article 20 data portability: collects 5 table exports for a user → JSON download
- Article 17 erasure: inserts `gdpr_requests` record with confirmation gate
- Requests list with status badges (pending/completed)

## New edge function

### `supabase/functions/security-admin/index.ts` (317 lines)
- `export-audit-log`: paginated query with filters, max 1000 records
- `list-sessions`: workspace member list with last_sign_in_at via admin client
- `revoke-session`: `auth.admin.signOut(userId, 'global')` + audit event
- `data-export-gdpr`: 5-table parallel export, updates gdpr_requests record

## DB changes (migration `20260512220000_payroll_security_platform.sql`)

- `enterprise_audit_events`: added `ip_address TEXT`, `user_agent TEXT` (IF NOT EXISTS)
- `data_retention_policies`: workspace_id, table_name, retention_days CHECK(>0), is_active DEFAULT false; UNIQUE(workspace_id, table_name)
- `gdpr_requests`: request_type CHECK(export/deletion), status CHECK(pending/in_progress/completed/rejected)
- `enforce_data_retention()`: SECURITY DEFINER function with safe allowlist (no dynamic SQL); pg_cron schedule NOT activated by this migration
