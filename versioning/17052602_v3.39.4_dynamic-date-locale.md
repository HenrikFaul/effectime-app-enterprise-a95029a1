## Problem

Every date-fns `format()` call and `<Calendar locale=â€¦>` prop in the enterprise app was hardcoded to Hungarian (`import { hu } from 'date-fns/locale'; â€¦ { locale: hu }`). Users with Czech, Slovak, Polish, German, Austrian, Romanian, or English locale setting were seeing Hungarian weekday abbreviations, month names, and date strings throughout the interface.

## Solution

Added `useDateLocale(): DateFnsLocale` hook to `src/i18n/I18nProvider.tsx` that maps the current app locale to the corresponding date-fns Locale object. All 24 affected files updated to use this hook instead of the hardcoded `hu` import.
