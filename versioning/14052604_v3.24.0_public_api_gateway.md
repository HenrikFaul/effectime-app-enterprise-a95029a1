## Two edge functions deployed

### `public-api`
Bearer-token authenticated REST gateway. Routes:
- `GET /v1/health` â€” connection test (returns workspace + key name)
- `GET /v1/employees` â€” active workspace members
- `GET /v1/schedules` â€” shift assignments
- `GET /v1/leave-requests` â€” leave requests

Auth flow:
1. Read `Authorization: Bearer <key>` or `X-API-Key: <key>`.
2. `sha256hex(rawKey)` matched against `enterprise_api_keys.key_hash`.
3. Reject if no row, expired, or rate-limit exceeded (1000 req/hour/key, in-memory sliding window).
4. Every request logged to `enterprise_api_usage_logs` with endpoint + method + status + response_ms.
5. Updates `enterprise_api_keys.last_used_at`.

Response envelope: `{ data: T[], meta: { request_id, count } }`.

### `webhook-dispatcher`
Drains `enterprise_webhook_deliveries` rows with status `pending` or `retrying`.
Up to 20 deliveries per invocation. Per delivery:
1. Build canonical envelope: `{ event, workspace_id, delivered_at, attempt, data }`.
2. Sign body with `HMAC-SHA256(subscription.secret, body)` â†’ `X-Effectime-Signature: sha256=<hex>`.
3. POST to subscription URL with `X-Effectime-Event` + `X-Effectime-Delivery-Id` headers.
4. 10-second timeout. On success â†’ `status='sent'`, `delivered_at=now()`. On failure â†’ `status='retrying'` until `attempt_count >= 3`, then `status='failed'`.

Platform-admin gated to prevent accidental DoS or spam from leaked credentials.
