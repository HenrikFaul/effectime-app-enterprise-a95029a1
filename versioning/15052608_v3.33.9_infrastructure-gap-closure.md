## Implementation details

### enterprise_coverage_rules VIEW
`AnalyticsDashboard.tsx` queries `enterprise_coverage_rules` for columns `(id, workspace_id, office_id, applies_to, min_headcount, business_role, skill_id)`. The underlying table is `enterprise_office_coverage_rules`. The view adds a derived `applies_to` column (`skill` when `skill_id` or `skill_ids` non-empty; `role` when `business_role` or `business_roles` non-empty; else `all`) and excludes archived rows (`archived_at IS NULL`). `SELECT` granted to `authenticated, anon`.

### email_queue table
`superadmin-hub` includes a try/catch soft-query for `email_queue` status counts. Without the table, every superadmin hub load logs a DB error. The table stores `recipient, subject, body_html, status (pending/sent/failed), timestamps`. RLS: only `user_roles.role = 'admin'` may SELECT.

### ai-copilot edge function
- Dual-client pattern (service_role admin + user JWT for auth check)
- `buildContextSummary()`: 4 parallel queries → member count, pending leave, upcoming shifts (90d), open compliance violations
- `getConversationHistory()`: last 10 messages from `ai_copilot_messages`, reversed to chronological order
- `storeMessage()`: inserts to `ai_copilot_messages` with structured_plan, model, token counts (non-blocking on failure)
- Graceful fallback: `ai_available: false` response when `ANTHROPIC_API_KEY` absent — UI never breaks
- Default model: `claude-sonnet-4-6`; caller may override via `model` field
- JSON parsing handles markdown fences; falls back to `{ raw, parse_failed: true }` on parse error

### create-workspace-user edge function
- Requires caller to be `owner` or `resourceAssistant` in the workspace
- Validates password via `validate_password_policy` RPC before creating auth user
- Paginated `listUsers` (perPage: 1000) to detect pre-existing emails
- Atomic rollback: deletes newly created auth user if membership insert fails
- Profile upserted non-blocking (warns on failure, does not abort)
- Returns `409` if user is already a member, `400` on password policy failure

### document-ai-polish edge function
- Auth + workspace membership check before loading document
- Graceful fallback when no `ANTHROPIC_API_KEY`: returns original HTML unchanged with hint
- Uses `claude-haiku-4-5-20251001` (fast, cost-effective for document tasks)
- System prompt instructs: preserve `{{double_braces}}` template variables exactly
- Persists polished HTML to `generated_documents.content_html`; warns but does not abort on DB update failure
