# v3.41.1 — Bug fixes: office-click crash, availability toggle, batch-fill office selector

**Date:** 2026-05-17  
**Branch:** claude/analyze-repo-structure-dE8F4  
**Type:** Bug fixes

## Summary

Three bugs introduced or surfaced after v3.41.0:

1. **White-screen crash on office name click (capacity planner)** — Root cause: `WorkspaceSettings` is a standalone top-level component function (not a closure inside `WorkspaceDashboard`), so it had no access to `highlightOfficeId` which was defined in `WorkspaceDashboard`. Added `highlightOfficeId?: string | null` to `WorkspaceSettings` props and wired it at the call site (line 558 in WorkspaceDashboard).

2. **Availability not editable on working days** — `canToggleAvail` required `!daySegs.length && !hasOncall`, blocking availability toggling on any day with recorded hours. Since pointer handlers already short-circuit when `!canEdit` (no DayEditorDialog risk), the guard was unnecessary. Simplified to `!canEdit && !!userId && !!membershipId`.

3. **Batch fill missing office selector** — User request: be able to assign a telephely to all selected days in one batch operation. `BatchFillDialog` now receives `offices`, `workspaceId`, `membershipId`, `userId`; a "Office / Site" select is shown when offices are available; after the segment loop, `upsertSiteAssignment` is called for each day if an office is selected.
