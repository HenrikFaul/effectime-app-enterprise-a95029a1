# v3.47.1 — Embed Snippet Builder token bug fix + ultra-premium UI refactor

**Date:** 2026-05-19
**Branch:** claude/analyze-repo-structure-dE8F4
**Type:** Bug fix + Enhancement

## Problem solved

Two issues reported by user:

1. **Critical bug**: The Snippet Builder in EmbedManager was inserting the token record's UUID (`id`) into generated iframe URLs instead of the raw 64-character hex token string. Every iframe generated via the builder was returning an authentication error.

2. **Visual quality**: All 5 embed views needed a design quality upgrade to match the premium, enterprise-grade aesthetic expected when displayed inside customer CRM systems. A detailed 34-section design brief was provided.

## Changes

### EmbedManager.tsx — Token bug fix
- Added `token: string` to `EmbedToken` interface
- Added `token` column to Supabase `.select()` query
- Changed `buildEmbedUrl(builderToken.id, ...)` → `buildEmbedUrl(builderToken.token, ...)`
- Renamed `buildEmbedUrl` first param from `tokenId` to `rawToken` for clarity
- Builder URL guard: `builderToken?.token` (falsy if token field missing)
