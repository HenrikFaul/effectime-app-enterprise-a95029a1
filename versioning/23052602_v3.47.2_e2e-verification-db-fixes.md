## Summary

End-to-end full-stack verification run against the entire effectime.app stack
(14 routes, 55+ RPCs, 143 tables). Two real bugs discovered and fixed:

1. **DB migration gap** — `v3_47_0_embed_new_views_and_rpcs.sql` existed on disk
   but had never been applied to the production database. Applied retroactively.

2. **`revoke_embed_token` silent failure** — the RPC returned `void` (no error)
   when a non-owner called it or the token didn't exist. The UPDATE silently
   matched zero rows. Fixed by adding `GET DIAGNOSTICS v_rows_updated = ROW_COUNT`
   and raising `P0001` if zero rows updated.

## Bug 1: Missing migration (v3.47.0)

**Detection:** Queried `supabase_migrations.schema_migrations` — latest entry was
v3_46_4. File `supabase/migrations/20260519100000_v3_47_0_embed_new_views_and_rpcs.sql`
existed in repo but was absent from the DB.

**Impact:** `check_member_availability` and `get_team_headcount` RPCs didn't exist
in production. Any CRM integration calling these would get a "function not found" error.

**Fix:** Applied via `apply_migration`. Verified both functions now exist with
`SECURITY DEFINER` and are callable by `anon`.

## Bug 2: revoke_embed_token silent failure

**Detection:** Called `SELECT revoke_embed_token(gen_random_uuid())` — expected
exception, received `void` with no error. Non-owner workspace members could "revoke"
tokens without any feedback or actual effect.

**Fix (migration 20260523100000_v3_47_1_fix_revoke_embed_token_silent_failure.sql):**

```sql
GET DIAGNOSTICS v_rows_updated = ROW_COUNT;
IF v_rows_updated = 0 THEN
  RAISE EXCEPTION 'Token not found or you are not authorised to revoke it';
END IF;
```

**Verification:** `SELECT revoke_embed_token(gen_random_uuid())` → raises
`P0001: Token not found or you are not authorised to revoke it`.

**Frontend compatibility:** `handleRevoke` in the embed token management UI already
checks `if (error) { toast.error(t('embed.toast_revoke_error')) }` — no frontend
change required. The new exception surfaces correctly.
