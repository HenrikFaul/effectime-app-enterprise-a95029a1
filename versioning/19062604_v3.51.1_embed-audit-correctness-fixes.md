# v3.51.1 — Embed: correctness fixes from a multi-agent adversarial audit

**Date:** 2026-06-19
**Branch:** claude/embed-assign-flicker-fix
**Type:** Bugfix (frontend correctness)

## Goal
Harden the embed module after the v3.50.1 (flicker fix) and v3.51.0 (Létszám removal + Munkatárs redesign) changes. A 5-dimension multi-agent audit (optimistic-writes, Munkatárs view, calendar timeline, wizard dialog, cross-cutting/regression) ran with every finding refuted by 3 independent verifiers; a second pass verified each fix. 13 of 15 findings confirmed and fixed; all frontend-only.

## Fixes (file → defect → fix)
1. **EmbedCapacityView / EmbedShiftRosterView / EmbedMemberScheduleView — optimistic `tmp:` removal (major).** Removing a still-optimistic shift sent a `tmp:<…>` id to `embed_remove_shift` (uuid param) → 22P02 error. Each remove handler now early-returns on `tmp:` ids; the remove control is disabled/hidden for un-persisted shifts.
2. **EmbedMemberScheduleView — workload summary precedence (major).** Counted shift days before holiday/leave, disagreeing with the cell (holiday/blocked > approved-leave > shift) and corrupting the week hours/overtime figure. Summary now mirrors the cell precedence (added `holidaySet` to deps).
3. **EmbedLeaveTimelineView — leave-type filter (major).** Options used `enterprise_leave_types` UUIDs while the data is the `leave_type` enum, so selecting any type hid all leaves. Options now use the enum keys with localized labels; added `timeline_view.legend_other` to all 8 locales; removed the now-unused `leaveTypes` memo.
4. **EmbedSmartScheduleDialog — coverage math (major).** Counted ALL office shifts (not rule-matched) as already-filled and indexed role slots by a raw count → under-scheduled multi-rule offices with wrong role composition. Now counts only rule-matched (role/skill) existing shifts via the same matcher as the grid's `shiftsFor`, and computes the still-open role slots greedily. Extended the dialog's `shift_assignments` type with `business_role`/`skill_id`.
5. **EmbedCapacityView — wizard finalize flicker (major).** `onCompleted={load}` did a non-silent reload (skeleton strobe). Now `onCompleted={() => load({ silent: true })}`.
6. **EmbedCapacityView / EmbedShiftRosterView / EmbedMemberScheduleView — write-clobber race (minor).** A stale silent reload issued before a newer write could overwrite the newer optimistic state during rapid editing. Added a `writeSeq` ref bumped per write and captured at load time; a silent reload whose captured value is stale is dropped (non-silent initial/navigation loads always apply).
7. **EmbedMemberScheduleView — activeOffice on member switch (minor).** The "Beosztás ide" office didn't re-default when switching member → silent wrong-office assignment risk. Now re-defaults to the new member's primary (tracked via `lastMemberId` ref) while keeping a manual choice for the same member.
8. **EmbedSmartScheduleDialog — partial-failure toast (minor).** Rendered success/total as a misleading range. New localized `smart_batch.save_failed_count` ("{{failed}}/{{total}} …") in all 8 locales.

## Correctly dismissed (0/3 verifiers)
- "Rejected RPC promise leaves savingKeys stuck" — `supabase.rpc().then` resolves with `{data,error}` (doesn't reject), so the key is always cleared.
- "Status filter offers rejected/cancelled that never match" — cosmetic, harmless.

## Known limitation (documented, unchanged)
Unchecking the wizard's "keep existing assignments" plans a full new headcount without removing prior assignees (upsert-only, no delete) — same as the native dialog; left as-is to avoid divergent half-behaviour.
