# v3.11.0 — Payroll Integration Engine

**Date:** 2026-05-12
**Branch:** claude/create-software-documentation-O7kj1
**Type:** New feature

## Scope

Multi-provider payroll period lifecycle management with client-side provider-specific CSV export and server-side aggregation edge function.

## Components delivered

### `src/components/enterprise/payroll/PayrollPanel.tsx` (958 lines)
- Props: `{ workspaceId: string; userId: string; isAdmin: boolean }`
- Period management: create (name + date range), list with Open/Locked/Exported status badges, delete
- Period summary: aggregates attendance hours + leave days + cost rates per member for selected period
- 4 KPI cards: Total Hours, Overtime Hours, Gross Estimate, Member Count
- Export dialog: 11-provider selector, client-side CSV generation, "Save as default provider" backed by `payroll_export_configs`
- Lock dialog: confirm → update status + write `payroll.period_locked` audit event
- All 51+ strings via `t('payroll.*')`

### Provider CSV formats
- **DATEV**: semicolon-delimited, German headers (Personalnummer;Name;Zeitraum;Normalstunden;Überstunden;Urlaubstage;Bruttolohn;Währung)
- **BambooHR**: comma, 6 English columns
- **Personio**: comma, 8 columns with period_start/end
- **All others (SAP, Workday, ADP, Sage, Billingo, Számlázz.hu, Pohoda, Generic)**: comma, 9 columns

## New edge function

### `supabase/functions/payroll-export/index.ts` (457 lines)
- `calculate-period`: aggregates hours/leave/cost from DB server-side
- `export-csv`: routes through provider formatters, updates `exported_at`/`exported_to`
- `lock-period`: validates open state, updates, writes audit event
- `export-api`: placeholder for future direct provider API integration

## DB changes (migration `20260512220000_payroll_security_platform.sql`)

- `payroll_periods`: id, workspace_id, name, start_date, end_date, status CHECK(open/locked/exported), locked_by, locked_at, exported_at, exported_to, created_by
- `payroll_export_configs`: workspace_id, provider, config JSONB, field_mappings JSONB, is_active; UNIQUE(workspace_id, provider)
- RLS: SELECT → is_enterprise_member; INSERT/UPDATE/DELETE → owner/resourceAssistant

- `supabase/config.toml`: registered `payroll-export` edge function with `verify_jwt = true`
