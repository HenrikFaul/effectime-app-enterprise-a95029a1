# v3.33.1 — Stabilization pass: data-integrity + audit-trail hardening

**Date:** 2026-05-14
**Branch:** `claude/review-project-requirements-lR0Wp`
**Scope:** Bug fixes only — no feature changes. Reconciles MCP-only schema
back to disk; closes documented gaps from a multi-phase audit.

## Why this release

A full system audit found that several schema objects added since v3.17.0
existed only in the remote DB (applied via Supabase MCP) and were never
committed to `supabase/migrations/`. A rebuild from disk would silently
regress the v3.17.1 strict-tier-key fix. The audit also surfaced:

- Tier-id immutability enforced only by convention.
- Two text[] columns (`tier_feature_keys`, `dependencies`) with no
  referential validation — typos silently hide UI permission slots.
- `tenant_feature_overrides.expires_at` not filtered on read.
- `tenant_subscriptions.(status, ends_at)` could be incoherent.
- A dead-code SQL-injection-shaped string in `run-report`.
- `send-scheduled-reports` marking partial failures as success.
- `cleanup-demo-workspace` returning ok=true on partial deletions.
- `superadmin-hub` performing privileged actions with no audit row.
- `payroll-export` silently emitting `display_name: 'Unknown'`.
- `useEnterprisePermissions` falling back to unfiltered catalog on
  legitimately-empty RPC results (defeating tier filtering).
- `InviteMemberDialog` useEffect with no cleanup; two hardcoded
  Hungarian placeholders; password fields leaking across mode toggle.

## What ships

### DB migration `v3_33_1_stabilization_reconciliation` (applied to remote)

1. `create_workspace_with_owner` — STRICT tier_key contract restored
   (raises on NULL / unknown; no silent freemium fallback).
2. `enforce_tier_id_immutability` trigger on `tenant_subscriptions` —
   blocks `UPDATE tier_id` outside the `superadmin_change_workspace_tier`
   RPC. The RPC sets `app.tier_change_rpc_active='true'` via
   `set_config(..., true)` (txn-local) inside its body; the trigger
   checks for that marker. `create_workspace_with_owner` also sets it
   before the initial tenant_subscriptions INSERT path.
3. `validate_tier_feature_keys` trigger on `enterprise_feature_catalog` —
   rejects newly-added array elements that don't exist in
   `features.feature_key`. Delta validation (only checks new keys) so
   pre-existing typos don't block unrelated UPDATEs.
4. `validate_feature_dependencies` trigger on `features` — same pattern
   for `features.dependencies`.
5. `tenant_feature_overrides_member_read` RLS policy now filters
   `expires_at > now() OR expires_at IS NULL` so expired overrides
   stop granting/revoking access without a cleanup job.
6. `tenant_subscriptions_status_ends_at_coherent` CHECK constraint
   (`NOT VALID`) — disallows future rows like `status='ended'` with
   NULL `ends_at`. NOT VALID protects existing rows; promote to VALID
   after one-time scrub.
7. `require_feature_id(_feature_key)` helper — raises if not found.
   Drop-in replacement for bare `(SELECT id FROM features WHERE
   feature_key='X')` subqueries in seeds.

### Edge function fixes

- **`superadmin-hub`** — every action now writes a `platform_audit_events`
  row with action `superadmin_hub.<action>`, actor_id, target_id, and
  body metadata. Fire-and-forget (logged on insert failure) to avoid
  bottlenecking the action.
- **`send-scheduled-reports`** — per-recipient try/catch. Marks the run
  as `success`, `partial_failure`, or `error` based on
  `failedCount / recipientsCount`. The `last_run_error` jsonb now
  carries the failed recipient list.
- **`cleanup-demo-workspace`** — returns `ok: false` and HTTP 207
  Multi-Status when at least one demo user delete fails; payload
  includes `failed_user_ids: string[]`.
- **`run-report`** — removed the dead-code raw-SQL `wrapped` template
  (workspace_id was string-concatenated, never executed; would have
  been an injection vector if exec_sql had been wired up). ORDER BY
  parsing now throws explicitly if the clause isn't a simple
  `column [ASC|DESC]` shape, instead of silently dropping it.
- **`payroll-export`** — when `display_name` falls back to 'Unknown',
  writes an `enterprise_audit_events` row
  (`payroll.export.member_profile_missing`) so ops can investigate.

### Frontend fixes

- **`useEnterprisePermissions`** — falls back to the legacy
  `enterprise_feature_catalog` SELECT ONLY when the RPC errored, not
  when it returned an empty array. A legitimately-empty result (e.g.
  freemium workspace with every UI slot tier-locked) is now correctly
  rendered as zero visible slots with the `hiddenByTier` notice
  instead of the unfiltered catalog. Also logs parallel-call errors
  per LESSON-CAPACITY-ERROR-001.
- **`InviteMemberDialog`** — useEffect now installs a `cancelled` flag
  + cleanup; every state setter post-await checks it. Switching
  invite ↔ create mode resets the password/confirm/show-password
  fields. Two Hungarian hardcoded placeholders (`"Kovács János"`,
  `"kolléga@cég.hu"`) replaced with i18n keys
  (`members.invite_name_placeholder`, `members.invite_email_placeholder`).
- **i18n** — 2 new keys × 8 locale files = 16 strings.
  Hungarian gets the original placeholders preserved; en/cs/sk/pl/de/at/ro
  get locale-appropriate values.

### Removed

- `CODEBASE_AUDIT_ROUND1.md` — file was corrupted (contained raw
  TypeScript instead of audit markdown); deleted.

## Verified false positives (no change)

- `Enterprise.tsx` `replace: true` uses (L92, L192, L216, L245, L257,
  L259) — all are intentional and inline-commented (legacy URL
  rewrite, auto-redirect loop prevention, one-time invite token).
- `InstallPwaPrompt` 30-day dismissal arithmetic — clock skew >30d
  just shows the prompt again, which is the right behavior.
- `RolePermissionManager` `refetch()` — `useEnterprisePermissions` is
  useState-based, not TanStack Query, so `invalidateQueries` doesn't
  apply.
- `is_tenant_member` STABLE — Postgres STABLE caches within a
  statement, not across statements/sessions; deactivating a member
  IS reflected on the next query.

## Manual QA matrix

   → must RAISE `tier_key is required`.

   admin → must RAISE `tier_id is immutable; use superadmin_change_workspace_tier()`.

   containing an unknown key → must RAISE `references unknown feature_key`.

   must be `partial_failure`, last_run_error must list the bad recipient.

   "Kovács János" (hu); switch to Create mode, type password, switch

   empty → tree should render empty with hidden-by-tier notice (not
   the unfiltered legacy catalog).
