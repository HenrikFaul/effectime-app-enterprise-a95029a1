# v3.31.0 — Candidate Scheduling / ATS bridge (Top-20 Rank 20)

**Date:** 2026-05-14
**Status before:** MISSING.
**Status after:** DONE for internal pipeline + public self-booking.

## What ships

- `candidates` table (status pipeline: new → screening → interview → offer → hired).
- `interview_slots` table with `booking_token` for public URL.
- `ats_integrations` table (provider config).
- Eligibility check rejects slots where any interviewer is on approved
  leave OR in another booked overlapping slot.
- 4 RPCs (3 manager-only + 1 public for self-booking).
- New routes: `/book/:token` (public, no auth) and recruiting panel
  inside the workspace Reports tab.
- 185 localized strings × 5 locales.

## Public self-booking flow

1. Manager creates a slot via the recruiting panel → DB generates a
   random `booking_token`.
2. Manager copies `${origin}/book/${token}` to clipboard and shares
   with candidate via email.
3. Candidate opens the URL → enters name + email → SECURITY DEFINER
   RPC `candidate_self_book` validates the token, upserts the
   candidate, marks the slot booked.
