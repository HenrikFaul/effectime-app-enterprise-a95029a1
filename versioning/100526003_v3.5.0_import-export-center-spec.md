## Cél

A jelenlegi korlátozott Settings-beli CSV import (`CsvImportPanel`) és egyszerű leave export (`ExportCenter`) helyett egy teljes, skálázható, entity-alapú, wizard-vezérelt Import/Export Center tervezési specifikációja — amely alapján a következő sprintben az implementáció elkészíthető.

## Specifikáció helye

`docs/IMPORT_EXPORT_CENTER_SPEC.md` — 20 fejezet, ~2000 sor részletes product + UX + technical spec.

## Főbb döntések (összefoglaló)

| Döntés | Ajánlás |
|--------|---------|
| Formátum | XLSX (elsődleges) + CSV (másodlagos); mindkettő elfogadott importnál |
| Entitás kezelés | Egy entitás egyszerre; config-driven entity registry |
| Mezőkijelölés | Exportnál admin választja a mezőket; required mezők zárva |
| Importálható mezők | Tagok: email, display_name, role, team, business_role, office_name, city, joined_at stb. |
| Védett mezők | user_id, profiles.email, created_at — soha nem importálható |
| Duplikátum kezelés | Partial success (sikeres sorok bekerülnek, hibások kihagyva + downloadable error report) |
| Upsert | Opt-in; alapértelmezés: csak új sorok |
| Template stabilitás | Machine key fejlécek + __schema_version rejtett oszlop + importAlias support |

## Érintett entitások (Phase 1–2)

- Tagok (Members)
- Szabadságok (Leave)
- Telephelyek (Locations)
- Munkakategóriák (Work Categories)
- Munkakörök (Job Roles)
- Pozíciók (Positions)
- Készségek (Skills)
