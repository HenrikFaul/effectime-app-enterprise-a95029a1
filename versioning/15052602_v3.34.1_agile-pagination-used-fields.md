# v3.34.1 — Agile backlog: auto-load, pagination, used-field detection

**Date:** 2026-05-15  
**Branch:** claude/architect-team-session-UWfpK  
**Type:** Enhancement (UX, Data visibility)

## Problem solved

The Backlog tab showed "Nincs adat" (No data) when no filter was applied, leaving users with a blank screen until they manually triggered a search. The Fields tab listed all discovered fields with no indication of which ones actually had data, making field selection a guessing game.

## Changes

### BacklogBrowser.tsx
- **Auto-load**: On mount, `loadFromCache()` fetches cached rows; if the cache is empty, automatically fires the default query (most recent items via `ORDER BY [System.ChangedDate] DESC` / `ORDER BY updated DESC`). The tab always shows data on open.
- **Pagination bar**: Dropdown at bottom with 10 / 20 / 50 / 100 / ALL options; default = 10. Item count shown alongside.
- `pageSizeRef` (useRef) ensures `search()` and `loadFromCache()` always see the current `pageSize` even in async callbacks (stale-closure prevention).

### FieldDiscovery.tsx
- **"Used" column**: Loads up to 200 cached `enterprise_agile_issues` rows; detects which mapped columns are non-null + which `issue_type` names appear. Displays a green CheckCircle (used) or grey Circle (not used).
- **"Show used only" toggle**: Filters table to used fields only.
- **"Select all used"**: One-click selects every field that has live data.
- Unused rows rendered at 70% opacity for visual clarity.

### i18n (all 8 locales)
Keys added: `backlog_browser.page_size_label`, `page_size_all`, `showing_count`, `field_discovery.col_used`, `btn_show_used_only`, `btn_show_all_fields`, `btn_select_all_used`, `used_hint`.

## Governance
- No new DB migrations (uses existing `enterprise_agile_issues` and `enterprise_workspace_integrations`).
- All 8 locale files updated in the same commit.
- No route changes; no tier logic touched.
