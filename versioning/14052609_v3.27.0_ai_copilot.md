# v3.27.0 — AI Scheduling Copilot (Top-20 Rank 1)

**Date:** 2026-05-14
**Status before:** MISSING (catalog ready).
**Status after:** DONE for single-call MVP; tool-use loop + Realtime streaming deferred.

## Highest-value feature in the Top-20 strategy

Per the strategy doc, AI-native scheduling commands a 2-3× ARR premium
over feature-equivalent non-AI products. This release ships the
conversational layer that turns Effectime into an "AI workforce
intelligence platform" instead of just a scheduling tool.

## Design: template-first-AI-second

Same pattern as v3.20 / v3.26: works without ANTHROPIC_API_KEY (returns
a configure-key fallback message), lights up with real Claude analysis
once the secret is configured. Operators don't have to wire up
Anthropic before deploying.

## DB

- `ai_copilot_conversations` — per-user session container.
- `ai_copilot_messages` — chat history with `structured_plan jsonb`,
  model name, input/output token counts.
- `ai_copilot_rate_limits` — 20 req/hour/user enforcement.
- `ai_copilot_start_conversation` RPC.

## Edge function (`ai-copilot`, v1 deployed)

1. Authenticate active workspace member.
2. Rate-limit check (returns 429 over 20/hour).
3. Eager-load aggregate context: member roles + locations + capacity,
   open leave next 90d, shift count next 90d, unresolved compliance
   violations.
4. Call Claude Sonnet 4.6 (default) with strict JSON output prompt.
5. Persist message + structured plan + record rate-limit hit.

**Privacy:** only counts/aggregates sent to Claude. No member names,
no PII. Per the strategy doc's privacy-first stance.

## Frontend

- `useAiCopilot.ts` hook + RPC helpers.
- `CopilotPanel` chat-style UI: message history, structured plan
  rendering (analysis + recommendations bullets + warning callout +
  confidence badge), Send button with loading state, token counter,
  privacy reminder.
- Wired inside the Calendar tab via FeatureGate (Enterprise tier only).
