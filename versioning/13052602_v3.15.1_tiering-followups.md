# v3.15.1 — Tiering follow-ups: audit viewer + full catalog localization

**Date:** 2026-05-13
**Branch:** `claude/review-project-requirements-lR0Wp` (PR #73)
**Scope:** Three items deferred from v3.15.0 as follow-up PRs, all delivered here.

## What changed

### 1. Audit log viewer UI (Superadmin Control Plane)

The v3.15.0 release added an immutable `platform_audit_events` table and made
`FeatureTiersTab` write to it on every tier/addon/routing change — but there
was no UI to read it back. This release adds the viewer.

- New file: `src/components/superadmin/PlatformAuditLogTab.tsx` —
  paginated table (50 rows / page) with:
  - Action filter (all 5 enum values + "all")
  - "Since" date filter
  - Client-side actor/target/action search
  - Actor display-name resolution via `profiles` batch join
  - Detail dialog rendering `prev_state` / `new_state` / `metadata` as
    side-by-side JSON
  - Color-coded action badges (green = enable, red = disable, blue =
    routing edit)
  - RLS error surfaced as a clear destructive card, not a crash.
- `SuperadminControlPlane.tsx` — new `'audit'` tab (ScrollText icon),
  registered with lazy mount alongside the existing `'tiers'` tab.

### 2. Tier and addon name localization

`tiers.<tier_key>.name` and `tiers.<tier_key>.description` plus
`addons.<addon_key>.{name,description}` keys added in all 5 locales
(en, hu, cs, sk, pl). Before this, Hungarian users saw English tier names
("Freemium") and English addon descriptions because the DB seed was the only
source.

- `src/lib/tiering/labels.ts` — new helper module with `tierName`,
  `tierDescription`, `addonName`, `addonDescription`, `featureName`,
  `featureDescription` functions. Each tries the i18n bundle first and falls
  back to the DB value if the lookup misses (translation acts as the source
  of truth where present, DB value is the safety net).
- Wired into `FeatureTiersTab` (tier dropdown, addon dropdown, tier filter
  badge) and `CreateWorkspaceDialog` (tier dropdown).

### 3. Feature catalog localization (135 features × 2 fields × EN+HU)

The 135 feature `name` and `description` strings were previously Hungarian-
only in the DB seed; foreign users saw raw Hungarian text in
`FeatureTiersTab` even though the surrounding chrome was localized. This
release adds:

- 135 EN translations of feature names + 1-line descriptions, hand-curated.
- HU bundle generated automatically from `docs/tiering/features.json` (the
  authoritative source).
- `scripts/build_feature_labels.mjs` — generator script with the EN
  translation table inline.
- `scripts/inject_feature_labels.mjs` — idempotent injector that writes the
  `features:` namespace block into both `en.ts` and `hu.ts` before the
  `platform_audit:` anchor.
- `FeatureTiersTab.tsx` — all feature label rendering switched to
  `featureName(t, key, dbName)` / `featureDescription(t, key, dbDesc)`. Covers
  the feature grid, the routing tree leaves, the FeatureNodeCard gradient
  cards, the dependency / dependents chips, and the FeatureDetailDialog
  title + description.
- cs/sk/pl deliberately *not* given per-feature names — the bundle-lookup
  fallback chain (active locale → en → key) keeps the UI functional in those
  languages with English feature labels, which is acceptable until a
  full-catalog translation pass is commissioned.

### 4. Platform-level i18n added for the audit viewer

- New `platform_audit.*` namespace in all 5 locales (action filter labels,
  table headers, detail dialog labels, pagination).
- `superadmin.tab_tiers` (replaces a hardcoded "Feature & Tier" string in the
  superadmin tab strip) and `superadmin.tab_audit` keys in all 5 locales.

## Known limitations / next steps

- `cs/sk/pl` per-feature translations are out of scope. The fallback chain
  shows English feature names to those users; if Hungarian fallback is
  preferred, swap the chain order. Full cs/sk/pl translations are a
  ~3-day translation procurement task documented in `docs_tasks.md`.
- The platform_audit viewer renders the action enum verbatim (e.g.
  `tier_feature_enabled`). A per-action localized label is an optional
  refinement.
- No CI hook yet for "DB seed and i18n names drifted" — if a future seed
  changes a feature name, the EN translation in build_feature_labels.mjs
  stays unchanged. Drift check is on the v3.15.x backlog.
