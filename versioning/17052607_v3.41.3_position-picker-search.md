# v3.41.3 — UX: Free-text search in position picker catalog

**Date:** 2026-05-17  
**Branch:** claude/analyze-repo-structure-dE8F4  
**Type:** UX improvement

## Problem solved

The "Előre definiált pozíció-katalógus" dialog showed a pure tree navigation (Category → Position → Skills). With 100+ positions across many categories, users had to click through multiple levels to find what they wanted. No way to search by text.

## Solution

Added a persistent free-text search `<Input>` (with magnifier icon) at the top of `PositionPickerDialog`. When a query is typed:
- All loaded roles are filtered client-side via `useMemo` (no extra DB round-trip)
- A flat list of matching roles is shown, each with a category subtitle
- Clicking a result: clears query, sets category, calls `handleSelectRole` → jumps directly to step 3 (skills review)
- Step breadcrumb hidden while search is active (not meaningful in flat-list mode)

When the query is cleared, normal tree navigation resumes.
