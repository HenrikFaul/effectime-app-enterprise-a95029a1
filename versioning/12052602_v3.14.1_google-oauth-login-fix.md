# v3.14.1 — Google OAuth Login Fix (HashRouter 404 flash)

**Date:** 2026-05-12
**Branch:** claude/add-multilingual-support-iWJLD

## Problem

After clicking "Sign in with Google", users saw a ~1-second 404 "Az oldal nem található" error screen. In some cases login became completely stuck with no way to proceed to the app.

## Root Cause

Supabase OAuth implicit flow delivers tokens in the URL hash fragment:

```
https://effectime.app/#access_token=eyJ...&refresh_token=...
```

The app uses `HashRouter` (react-router-dom), which treats everything after `#` as the client-side route path. The string `access_token=eyJ...` matched no route → the `path="*"` catch-all rendered `<NotFound />`.

After `setSession()` resolved in `useAuth.tsx`, the code called `window.history.replaceState()` to strip the tokens from the URL. **`replaceState` does not fire a `hashchange` event**, so HashRouter never re-evaluated its routes and remained stuck on the 404 page even though the session was now valid.

## Fix

### `src/hooks/useAuth.tsx`

Replaced `setSession(...).finally(() => replaceState(...))` with:

```tsx
supabase.auth.setSession({ access_token, refresh_token })
  .then(({ error }) => {
    if (!error) {
      window.location.replace(`${window.location.origin}/#/app`);
    } else {
      window.location.replace(`${window.location.origin}/#/auth`);
    }
  });
```

`window.location.replace()` fires a `hashchange` event, which HashRouter listens to and uses to re-evaluate routes. On success the user lands on `/app`; on error they are sent to `/auth` to see the error message.

### `src/App.tsx`

Added `OAuthCallbackGuard` component that synchronously detects `#access_token=` in the initial hash (via `useState` lazy initializer) and renders a loading spinner instead of the `HashRouter`. This fully prevents the 404 flash — the router is simply not mounted while the OAuth tokens are present.

Once `useAuth` calls `window.location.replace(/#/app)`, the guard's `hashchange` listener clears the flag, `HashRouter` mounts fresh with the clean `#/app` URL, and the user lands on the app.

## Files Changed

- `src/hooks/useAuth.tsx` — setSession callback: `replaceState` → `location.replace`
- `src/App.tsx` — added `OAuthCallbackGuard`; `<HashRouter>` wrapped with it; `useState` added to React import
