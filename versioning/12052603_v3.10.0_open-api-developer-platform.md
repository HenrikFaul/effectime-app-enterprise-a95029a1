## Scope

Self-service API key management, webhook subscriptions, and developer portal UI for integrating Effectime with external tools.

## Components delivered

### `src/components/enterprise/developer/DeveloperPortal.tsx`
- Props: `{ workspaceId: string; userId: string }`
- **API Keys tab:**
  - Create named keys with scope selection (read, write, webhooks, admin)
  - Key format: `eff_<workspaceId[0:8]>_<24-char-uuid>`
  - SHA-256 hash stored via `crypto.subtle.digest` — raw key shown once in dialog, never re-readable
  - List active keys with last-used timestamp and scope badges
  - Revoke key (sets `revoked_at`)
  - 7-day usage BarChart (Recharts) from `enterprise_api_usage_logs`
- **Webhooks tab:**
  - Create subscription: HTTPS-only URL validation, auto-generated signing secret, event type checkboxes (member.created, member.updated, leave.requested, leave.approved, leave.rejected)
  - Toggle active/inactive, delete subscription
  - Shows last_fired_at and last_error per subscription
- **Developer reference card:** authentication header format, rate limit, endpoint overview

## DB changes

New tables (migration `supabase/migrations/20260512210000_analytics_wellbeing_api_platform.sql`):
- `enterprise_api_keys` — workspace_id, created_by, name, key_prefix, key_hash, scopes TEXT[], last_used_at, expires_at, revoked_at
- `enterprise_api_usage_logs` — workspace_id, api_key_id, method, path, status_code, duration_ms
- `enterprise_webhook_subscriptions` — workspace_id, created_by, url, secret, events TEXT[], is_active, last_fired_at, last_error; updated_at trigger

RLS: members SELECT own, admins ALL.

## Security notes

- API key raw value shown ONCE via modal and never stored in plaintext
- Only SHA-256 hash stored in DB
- Webhook URLs validated for HTTPS scheme before save
- All tables have RLS; non-admin members cannot see API keys or webhooks of others
