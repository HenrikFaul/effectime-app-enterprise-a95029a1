# v3.40.0 — Structured Open Shifts: position FK + multi-skill + top-3 candidates + escalation + waitlist

**Date:** 2026-05-17  
**Branch:** claude/analyze-repo-structure-dE8F4  
**Type:** Feature / Infrastructure

## Problem

Open shifts were broadcast with a free-text "Szükséges pozíció" input. There was no structured link to the role catalog, no eligibility-based candidate pre-selection, no configurable escalation timeout, no waitlist for filled shifts, and no automated replacement search when an employee cancelled their assignment.

## Solution

End-to-end structured staffing workflow: manager selects a position from the workspace role catalog (via `PositionPickerDialog`), optionally picks skills, sees the top-3 eligible candidates ranked by the existing `coverageEligibility.ts` engine, and chooses between notifying only those candidates or broadcasting to all matching members. The DB handles escalation automatically via pg_cron every 15 minutes. Employees can join a waitlist for filled shifts; the replacement RPC automatically promotes the first waitlisted person when an assignment is cancelled.

## Technical details

### Migration: `20260517200000_v3_40_0_structured_open_shifts.sql`

**Schema changes (backward-compatible):**
- `enterprise_open_shift_requests`: `role_id uuid FK enterprise_workspace_roles`, `skill_ids uuid[] DEFAULT '{}'`, `respond_by_at timestamptz`, `escalation_level int DEFAULT 0`, `notified_user_ids uuid[] DEFAULT '{}'`, `target_user_ids uuid[] DEFAULT '{}'`, `timeout_hours numeric DEFAULT 3`
- New `enterprise_open_shift_waitlist`: id, request_id FK, membership_id, user_id FK, position int, status CHECK('waiting'|'notified'|'assigned'|'dropped'), UNIQUE(request_id, user_id)
- New `enterprise_shift_cancellations`: audit trail with replacement tracking fields

**RPCs:**
- `create_open_shift_request` updated: resolves role name from FK, merges legacy skill_id, sets respond_by_at, filters notifications to role/skill-matching members only (or exact `target_user_ids`)
- New `join_open_shift_waitlist(_request_id)` → jsonb `{ok, position}`
- New `cancel_shift_assignment(_assignment_id)` → deletes assignment, re-opens or creates replacement open shift, promotes first waitlisted member, notifies managers if no replacement found
- New `process_open_shift_escalations()` → pg_cron `*/15 * * * *`, sends next 5-member batch to expired unfilled requests (up to escalation_level 5)

### Hooks (`src/hooks/useOpenShifts.ts`)
- `OpenShiftRequest` type: added `role_id`, `skill_ids`, `respond_by_at`, `timeout_hours` fields
- `useCreateOpenShift`: now accepts `roleId?`, `skillIds?`, `timeoutHours?`, `targetUserIds?`
- New `useJoinWaitlist()`
- New `useCancelShiftAssignment()`
- New `useShiftCandidates({ workspaceId, officeId, shiftDate, businessRole, skillIds, enabled })` — parallel fetch of members + skills + assignments + leaves + holidays + blocked dates, runs `rankCandidates()`, returns sorted `EligibilityResult[]`

### Components
- `OpenShiftManager.tsx`: `PositionPickerDialog` replaces free-text role input; top-3 candidate panel with checkboxes; dual-action buttons ("Notify selected N" / "Broadcast to all matching"); configurable `timeout_hours` input; `skill_ids[]` badge rendering
- `OpenShiftPanel.tsx`: waitlist button for filled shifts; filter updated to match against `skill_ids[]` array with legacy `skill_id` fallback
