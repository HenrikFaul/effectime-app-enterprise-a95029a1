## Feature behaviour

### BacklogFilterBuilder (visual mode)
- **Work item type / Issue type**: multi-checkbox list derived from `ado.workitemtype.*` metadata (ADO) or `jira.issuetypes` schema (Jira)
- **State / Status**: multi-checkbox list aggregated from all work item type `schema.states` arrays
- **Assigned to**: project member checkboxes, loaded once via `search_assignable_users` proxy action (ADO vssps identity API; Jira team members)
- **Iteration path** (ADO only): dropdown from `ado.iterations` schema.paths
- **Created date range**: two date-type inputs â†’ `System.CreatedDate >= / <=` (ADO) or `created >= / <=` (Jira)
- **Text search**: title CONTAINS (ADO) / summary ~ (Jira)
- Active filter badges with one-click removal
- Assembles valid WIQL / JQL and calls `search_issues` proxy on "Search"

### FieldDiscovery (field selection)
- "Board" checkbox column added to the field table
- Selection initialised from `integration.selected_field_ids`
- "Save selection" button persists via `UPDATE enterprise_workspace_integrations SET selected_field_ids = [...]`
- Shows `N selected` counter

### BacklogBrowser mode toggle
- Pill-style "Visual filter" / "WIQL" toggle in card header
- Default: visual mode
- WIQL mode preserves all existing presets and manual query input

## Non-regressions
- Existing WIQL/JQL mode preserved exactly as before
- `search()` signature unchanged for Jira callers; optional `queryOverride` param added
- All i18n keys follow established namespace pattern
