# v3.51.2 — Native calendar: fix the leave-type filter (enum vs UUID)

**Date:** 2026-06-19
**Branch:** claude/embed-assign-flicker-fix (folded into PR #165)
**Type:** Bugfix (frontend)

## Goal
Fix the same leave-type filter defect in the native calendar that the v3.51.1 multi-agent audit found in the embed twin. `src/components/enterprise/calendar/TimelineView.tsx` built the "Szabadság típusa" filter options from `enterprise_leave_types` rows (`{ value: id (UUID), label: name }`), but the data field `leave_requests.leave_type` is the `public.leave_type` ENUM (`vacation | sick_leave | unpaid_leave | other`). The match (`filters.leave_type.includes(l.leave_type)` by UUID + a `name === enum` fallback that never holds) meant selecting ANY leave type hid every leave.

## Fix
`TimelineView.tsx`:
- `filterOptions.leave_type` now uses the enum keys with localized labels: `vacation → t('timeline_view.legend_vacation')`, `sick_leave → legend_sick`, `unpaid_leave → legend_unpaid`, `other → legend_other`.
- `leavesByUser` match simplified to `filters.leave_type.length === 0 || filters.leave_type.includes(l.leave_type)`.
- No new i18n strings — reuses the `legend_*` keys (incl. `legend_other` added to all 8 locales in v3.51.1). `leaveTypes` state/fetch left intact (still referenced by the memo deps; harmless under `noUnusedLocals: false`).

## Placement note
Folded into PR #165 (branch `claude/embed-assign-flicker-fix`) rather than a separate branch off `origin/main`, because the fix depends on `timeline_view.legend_other`, which only exists on that branch (added in v3.51.1, not yet on `main`). A separate off-`main` PR would duplicate that key (locale-file conflict with #165) and scramble the CHANGELOG version ordering. It is the native counterpart of the embed calendar fix already in this PR.

## Acceptance criteria
- [x] Selecting a leave type in the native calendar shows matching leaves (instead of hiding all).
- [x] Colour tints and other filters unchanged; no new strings.
- [x] `tsc --noEmit` ✓; `vite build` ✓.

## Non-regression
- Frontend-only; no RPC/schema change. `LEAVE_TYPE_TINT` already keyed on the enum, so rendering is unaffected.
