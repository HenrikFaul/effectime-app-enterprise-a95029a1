## Implementation details

### adoSyncProjectConfig (replaces stub)

Three-step sync:
1. `GET /_apis/wit/workitemtypes?api-version=7.0` — all work item types with their `states[]` array; stored as `ado.workitemtype.{name}` rows in `enterprise_agile_field_metadata`
2. `GET /_apis/wit/classificationnodes/Iterations?$depth=10` — full iteration tree; flattened to path list, stored as single `ado.iterations` row with `schema.paths[]`
3. `GET /_apis/wit/fields?api-version=7.0` — all project fields; stored as individual rows (same as before)

### adoGetIssue

`GET /_apis/wit/workitems/{id}?$expand=All&api-version=7.0`
Maps all core system fields: Title, Description, State, AssignedTo (email + displayName), WorkItemType, Priority, IterationPath, AreaPath, StoryPoints, OriginalEstimate, RemainingWork, Parent, CreatedDate, ChangedDate, Tags.

### adoGetStates

`GET /_apis/wit/workitemtypes/{type}?api-version=7.0`
Returns states array from the type definition. Used by `get_issue` and `get_transitions` router cases.

### adoSearchIdentities (two-strategy)

1. Try `https://vssps.dev.azure.com/{org}/_apis/identities?searchFilter=General&filterValue={q}` — covers AAD + MSA users
2. Fall back to `GET /_apis/projects/{proj}/teams` → `GET /_apis/projects/{proj}/teams/{id}/members` — project member enumeration

### adoUpdate (expanded)

Now handles: title, description, assignee_email (null = unassign), iteration_path (null = clear), state, priority (numeric 1–4 for `Microsoft.VSTS.Common.Priority`), story_points.
Changed `if (payload.field)` guards to `if (payload.field != null)` so empty string and 0 are handled correctly.

### AzureDevOpsIssueEditor

- Opens via `get_issue` action (same as Jira); receives `{ issue, transitions: states[] }`
- State selector: dropdown populated from `transitions` returned by server (ADO states, not Jira transitions)
- Assignee: text search field + select from returned identities; email is the update value (ADO accepts email for `System.AssignedTo`)
- Iteration path: text input with `<datalist>` from cached `ado.iterations` metadata
- Priority: select from 1=Critical … 4=Low
- Save: `update_issue` action; no-op if no changes detected
- Full metadata row below the form: type, iteration, area, parent ID, created, updated
