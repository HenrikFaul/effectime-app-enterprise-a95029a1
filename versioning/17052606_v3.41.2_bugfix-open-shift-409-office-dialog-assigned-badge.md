## Bugs fixed

### 1. create_open_shift_request → HTTP 409 Conflict (CRITICAL)

**Root cause:** PostgreSQL had two overloads of `create_open_shift_request`:
- 6-param: `(uuid,uuid,date,text,uuid,text)` — from migration v3.39.0
- 10-param: `(uuid,uuid,date,text,uuid,text,uuid,uuid[],numeric,uuid[])` — from v3.40.0

Because ALL parameters have DEFAULT values in BOTH functions, PostgreSQL / PostgREST cannot disambiguate the call → "ambiguous function call" → HTTP 409 Conflict.

**Fix:** Migration `20260517220000_v3_41_2_drop_old_create_open_shift_overload.sql` drops the old 6-param overload. Applied directly to Supabase project `oezlzzmzzvbvinuysxaz`.

**Lesson added:** HIBA-079 in `codingLessonsLearnt.md`.

### 2. OfficeEditorDialog too narrow

The dialog was `max-w-2xl` — office form fields were cramped and some content overflowed. Changed to `max-w-3xl w-full`.

### 3. Employee calendar missing "Beosztva" badge

`enterprise_shift_assignments` entries are loaded as `siteAssignments` but were only surfaced as a small office name when `displayConfig.site` was ON. Employees couldn't clearly see they had a formal shift assignment.

**Fix:** Added a dedicated orange `attendance.assigned_badge` label that always appears when `siteForDay` exists, regardless of display config. The office name (from `displayConfig.site`) still shows below it.
