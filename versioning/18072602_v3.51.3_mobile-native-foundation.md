Date: 2026-07-18

Follow-up: [`18072603_v3.51.3_mobile-secure-storage-csp.md`](18072603_v3.51.3_mobile-secure-storage-csp.md)

## Requirement

**BIZONYÍTOTT:** Android and iOS application development must be possible from
the existing Effectime product, and both native clients must use the same data
sources as the web application. The current user request is the primary source;
the repository's mobile strategy also documents native store distribution,
deep links, offline, push and biometric follow-up requirements.

## Implemented foundation

- One React/Vite bundle for web, Android and iOS.
- One public Supabase configuration and the existing Auth, PostgreSQL/RLS, RPC
  and Edge Function trust boundary for all three clients.
- Exact-pinned Capacitor 8 runtime, CLI, Android/iOS platforms, App and Browser
  plugins; Node.js 22.9+ contract and reproducible mobile scripts.
- Production-safe Capacitor configuration without a remote preview URL or
  cleartext server override.
- Version-controlled Android API 24+/target 36 and iOS 15+ projects using the
  proposed `app.effectime` identity.
- PKCE-only native auth through the system browser, cold/warm callback handling,
  lifecycle-aware refresh and verified recovery intent. Implicit token pairs
  supplied through a custom scheme are rejected.
- Canonical `https://effectime.app` public-link construction, native PWA/SW
  separation, auth URL log/cache hardening and Android backup/cleartext denial.
- Fail-closed mobile contract checks, component behavior tests and complete
  operational handoff in [`docs/mobile/README.md`](../docs/mobile/README.md).

No backend schema or API contract changed. Older web callback behavior remains
available only in the web runtime.

## Release decision

**Foundation GO; store release NO-GO.** Before signed distribution:

1. confirm and reserve `app.effectime` in Play Console and App Store Connect;
2. validate the implemented Keychain/Keystore-backed Supabase session storage,
   migration/reinstall/logout and recovery paths on physical devices;
3. validate the implemented native WebView CSP on Android WebView and iOS
   WKWebView with zero unexpected violations;
4. configure exact Supabase mobile redirects and verified HTTPS App/Universal
   Links using real Apple Team and Android signing data;
5. replace Capacitor template icon/splash files with approved Effectime assets;
6. obtain green hosted evidence from the existing Android and macOS/Xcode compile
   CI jobs, then add emulator/simulator smoke tests;
7. pass physical Android/iOS auth, tenant/RBAC, lifecycle and critical-flow smoke;
8. release through Play Internal Testing and TestFlight first.

Push notifications, a 14-day encrypted offline data model and biometric unlock
are separate product/security packages. None is claimed by this foundation.
