## Design Concept: GiGanttIc

### Brand Identity
- **Name**: "GiGanttIc" — typographically rendered as `[Gi][Gantt][Ic]` where "Gi" and italic "Ic" carry teal accent color. The full name reads as both "gigantic" (scale ambition) and "Gi+Gantt+Ic" (product brand).
- **Icon**: `Sparkles` — signals premium, flagship status.
- **Tab**: `data-[state=active]:bg-teal-500/15` treatment with branded typography — distinct from standard tab active states.
- **Badge**: "BOARD" pill in teal for board identity.

### Visual Tone
- **Surface**: `#0d0f14` (deep obsidian) with `#12151d` toolbar, `#0a0c10` status bar — layered depth.
- **Borders**: `border-white/8` baseline, `border-white/10` separators — crisp but subtle.
- **Accents**: teal `rgba(20,184,166,*)` for today marker, hover states, selection, dependency lines, progress fills.
- **Bar colors**: type-coded palette — violet (Epic), teal (Story), sky (Task), red (Bug/Overdue), amber (Sub-task), gold diamond (Milestone), emerald (Done).
- **Typography**: 9–11px tight tracking; uppercase 8px labels for section headers.

### Layout Architecture
- **Single `overflow: auto` container** — eliminates JS scroll sync entirely.
- Left task column: `position: sticky; left: 0; z-index: 10` — stays anchored during horizontal scroll.
- Timeline header row: `position: sticky; top: 0; z-index: 20` — stays anchored during vertical scroll.
- Top-left intersection: `position: sticky; top: 0; left: 0; z-index: 30`.
- Content width: `LEFT_W (292px) + chartWidth (totalDays × dayPx)`.

## Component Architecture

### GiGanttIcBoard (main export)
**Props**: `{ issues: GiGanttIssueRow[]; onOpen?: (key: string) => void }`

**State**:
- `zoom: ZoomMode` — `'week' | 'month' | 'quarter'`
- `expandedKeys: Set<string>` — initially auto-expands all Epics
- `selectedKey: string | null` — drives DetailsPanel
- `hoveredKey: string | null` — row highlight sync
- `search: string`, `typeFilter: string` — filtering
- `fullscreen: boolean` — fixed viewport overlay

**Memos** (all derived, no mutation):
- `filtered` — search + type filter applied
- `byKey, roots` — tree from `buildTree()` + `getRoots()`
- `visibleRows` — flat list from `flattenTree()` respecting `expandedKeys`
- `viewStart, totalDays` — from `computeTimeline()`
- `chartWidth` — `totalDays * dayPx`
- `primarySegs, secondarySegs` — header column definitions
- `todayX` — pixel offset of today
- `depLines` — SVG path data for parent→child connectors

### DetailsPanel (inline)
Opens on row click. Shows: type badge, status, overdue flag, progress bar, schedule fields, effort fields, assignee avatar, hierarchy tree (parent + children), labels, capacity risk alert. "Open in Jira" button if `onOpen` provided.

### GiGanttEmptyState / GiGanttLoadingState (exports)
Premium branded empty/loading states for parent use.

## Tree & Timeline Logic

### Hierarchy (buildTree)
1. `Map<external_key, GanttNode>` from all issues
2. For each node: if `parent_key` exists in map → attach as child; else → root
3. Sort siblings by `start_date ?? due_date`
4. `flattenTree(roots, expandedKeys)` produces ordered `VisibleRow[]`

### Timeline (computeTimeline)
- `viewStart` = `min(all dates) − PAD_DAYS[zoom]`, clamped to at most 1 year ago
- `viewEnd` = `max(all dates) + PAD_DAYS[zoom]`, at minimum `viewStart + MIN_SPAN_DAYS[zoom]`
- `totalDays = daysBetween(viewStart, viewEnd) + 1`
- `chartWidth = totalDays × DAY_PX[zoom]`

### Header Segments
- **Week zoom**: primary = months, secondary = ISO weeks (Monday-aligned)
- **Month zoom**: primary = quarters (Q1–Q4 year), secondary = month abbreviations
- **Quarter zoom**: primary = years, secondary = month abbreviations

### Dependency SVG
- Iterates `visibleRows` to build `rowIdx: Map<key, index>`
- For each row with `parent_key` in `rowIdx`: compute bar positions → Bézier S-curve
- `M x1,y1 C midX,y1 midX,y2 x2,y2` with `marker-end="url(#gg-arrow)"`

## Backend Schema

### enterprise_agile_issues (additive columns)
| Column | Type | Purpose |
|--------|------|---------|
| `is_milestone` | boolean | Renders as gold diamond in GiGanttIc |
| `progress_pct` | numeric 0–100 | Manual override for progress fill |
| `dependency_keys` | text[] | Explicit FS deps beyond parent_key |
| `critical_path` | boolean | CPM flag for future algorithm |
| `gantt_color` | text | Per-task color override |
| `gantt_row_order` | integer | Manual sort within group |

### enterprise_ganttIc_dependencies (new table)
- `predecessor_key`, `successor_key` (external_key references)
- `dep_type`: FS / SS / FF / SF
- `lag_days`: positive = lag, negative = lead
- `is_auto`: true when derived from parent_key
- Unique on `(integration_id, predecessor_key, successor_key)`
- RLS: read = all members; write = owner + resourceAssistant

### ganttIc_has_dependency_cycle()
- Recursive CTE BFS: starting from `successor`, traverse all successors; return true if `predecessor` is reachable
- SECURITY DEFINER + `SET search_path = public, pg_temp`
- Must be called before any INSERT into `enterprise_ganttIc_dependencies`

## Performance Considerations

- **No virtualization**: With typical 33–200 issues, direct rendering is fast. All rows are DOM elements.
- **Fixed row height (40px)**: predictable layout, no dynamic height recalculation.
- **CSS sticky positioning**: zero JS scroll overhead, browser-native GPU-accelerated sticky.
- **SVG overlay**: single SVG for all dependency curves + today marker; `pointer-events: none` — no interaction cost.
- **Memoization**: all expensive derivations (tree, timeline, segs, depLines) wrapped in `useMemo` with correct dependency arrays.
- **Future**: row virtualization via `react-virtual` or `@tanstack/react-virtual` if row count exceeds ~500.

## Extension Points

1. **Critical Path Algorithm**: `ganttIc_has_dependency_cycle` + `enterprise_ganttIc_dependencies` provide the foundation. Next: topological sort → forward/backward pass → critical path flag → red bar highlighting.
2. **Drag-to-move bars**: Add `@dnd-kit/core` or native pointer events for date-change interactions.
3. **Baseline comparison**: Add `baseline_start_date / baseline_end_date` columns to compare planned vs actual.
4. **Export**: SVG/PNG export of the visible chart area.
5. **Row virtualization**: `@tanstack/react-virtual` for >500 row datasets.
6. **Resource histogram**: Below-chart swimlane showing assignee workload per day.
7. **Writeback**: Update `start_date / due_date` via Jira API when user drags bars.
