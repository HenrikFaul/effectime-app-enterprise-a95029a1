# v3.51.0 — Embed: remove redundant "Létszám" + redesign "Menetrend" → "Munkatárs"

**Date:** 2026-06-19
**Branch:** claude/embed-assign-flicker-fix
**Type:** Feature (UX simplification + view redesign)

## Goals
From the customer's review of the embed tabs:
1. Remove the **Létszám** (office_headcount) view — redundant with **Kapacitás**.
2. Replace the unclear, clunky **Menetrend** (member_schedule) with a genuinely better per-person scheduling function: smarter, more efficient, more understandable.

## Why Létszám was redundant
It was a read-only office × day roll-up of `assigned / required` (required = Σ of the office's coverage-rule minimums). **Kapacitás already shows the same numbers per rule** (the office total is just the sum) *and* supports editing + monthly view. The only thing lost is a per-office aggregate glance, inferable from Kapacitás.

## "Munkatárs" redesign — what changed vs the old Menetrend
| Old (Menetrend) | New (Munkatárs) |
|---|---|
| Separate full-screen member **picker gate**, then the schedule | **Inline member dropdown** in the header — one click to switch |
| Single flat **week** (7 cards) | **Month calendar** grid + **Heti/Havi** toggle + prev/next + "Ma" |
| No workload context | **Summary**: scheduled days · leave days · (week) hours vs weekly capacity, flags overtime |
| Assign silently dumped into the member's primary office | **"Beosztás ide: <telephely>"** selector — explicit target office |
| Full reload on each write (flicker) | **Optimistic + silent** writes (same pattern as v3.50.1) |
| Unclear tab name "Menetrend" | Clear tab name **"Munkatárs"** |

## Implementation notes
- The redesigned view self-fetches per the visible period via `get_embed_view_data` (view `member_schedule`) and reads the enriched member fields (`weekly_capacity_hours`, `base_working_hours`) added in v3.50.0 for the workload summary.
- Member switching is local (no refetch) — all members' shifts for the period are already in the payload. Only period/range changes refetch (with skeleton); writes refetch silently.
- The month grid renders `eachDayOfInterval(startOfWeek(startOfMonth)…endOfWeek(endOfMonth))` chunked into rows of 7; out-of-month days are dimmed and non-interactive.
