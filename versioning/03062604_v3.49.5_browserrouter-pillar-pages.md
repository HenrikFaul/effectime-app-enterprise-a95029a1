## Changes

### Router migration
- `src/App.tsx`: `HashRouter` → `BrowserRouter`. Inverted `HashRouteBridge`: old `/#/auth?ws=1` bookmarks now redirect to `/auth?ws=1`. Supabase OAuth fragments (`#access_token=…`) untouched.
- `vercel.json` + `public/_redirects` already had SPA fallback — no infra change needed.
- `HASH_ROUTE_QUERY_KEYS` constant removed (no longer needed).

### SEO infrastructure
- `bun add react-helmet-async`.
- `src/main.tsx`: wrap `<App />` in `<HelmetProvider>`.
- `src/components/seo/SeoHead.tsx` (new): per-route head primitive — title, description, canonical, hreflang, OG, Twitter, BreadcrumbList + arbitrary extra JSON-LD.
- `src/components/seo/PillarPageLayout.tsx` (new): shared semantic layout for pillar pages (H1 + sections + FAQ + breadcrumb nav + related links). Tokens-only styling.
- `index.html`: static `<link rel="canonical">` removed — per-route Helmet owns it now. Prevents duplicate-canonical regression on pillar pages.
- `src/pages/Landing.tsx`: now uses `<SeoHead />` with `path="/"` so `/` keeps a canonical after the static one was removed.

## Anti-regression

- **OAuth callback unchanged**: `OAuthCallbackGuard` already checks `window.location.hash` for `access_token=`/`refresh_token=`. BrowserRouter does not consume the hash. Supabase callbacks continue to work end-to-end.
- **PWA / 404 SPA fallback**: `public/404.html` writes to `sessionStorage.spa_redirect` then redirects to `/`; `SpaRedirectHandler` reads and navigates. Compatible with BrowserRouter (better than before: no more hash-encoded path).
- **Existing deep links**: `/#/auth?ws=…` style bookmarks now transparently rewrite to `/auth?ws=…` via the inverted `HashRouteBridge`.
- **Canonical duplication risk**: eliminated — static canonical removed; each route has exactly one Helmet-emitted canonical (or none, which Google self-canonicalizes).
