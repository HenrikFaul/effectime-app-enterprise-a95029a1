## 1. Jira proxy bugfix

**Symptom**: After the Jira tenant migrated to the new search API
(`/rest/api/3/search/jql`, May 2025), the existing proxy occasionally returned
zero results or 410 Gone, breaking every downstream feature (Backlog browser,
Capacity Fit, writeback). The Atlassian `/myself` test continued to succeed,
masking the real failure point.

**Root cause**: `jiraSearch()` requested only a fixed list of system fields and
relied on a single endpoint. Tenants that respond to `GET` only, or still on
the legacy search route, fell off the cliff.

**Fix** (`supabase/functions/jira-devops-proxy/index.ts`):
1. Request `fields: ['*all']` so Jira returns every navigable + custom field.
2. Cascade three endpoints: `POST /search/jql` → `GET /search/jql` →
   `GET /search` (legacy). First 2xx wins.
3. Map description (Atlassian Document Format) into plain text via a recursive
   `adfToText()` walker.
4. Resolve `sprint`, `team`, `start_date`, `story_points` across the most
   common Jira customfield IDs (`customfield_10020 / 10007 / 10010 / 10001 /
   10015 / 10016 / 10026`).

## 3. New `AgileBoards` component

File: `src/components/enterprise/agile/AgileBoards.tsx`
Wired into `src/components/enterprise/agile/AgilePanel.tsx` as a new tab
(`Boards`), positioned between *Backlog* and *Issue írás*.

Three sub-views in a `Tabs` shell:

- **Kanban** — Trello-style columns by `status`, ordered with the canonical
  sequence (To Do → In Progress → In Review → Done).
- **Scrum** — Sprint-grouped lanes with status sub-columns, header summary of
  ticket count + Story Points.
- **Gantt** — Month-axis horizontal bars driven by `start_date → due_date`,
  coloured by `issue_type` (Bug=red, Epic=purple, Story=emerald,
  Task/other=sky). Tickets without dates are skipped (with empty-state hint).

All three views read directly from the local cache (`enterprise_agile_issues`)
so they keep working offline. A *Szinkron* button on the card header triggers a
fresh `search_issues` call which both refreshes the table and the view.
