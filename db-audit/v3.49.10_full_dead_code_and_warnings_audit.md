## 6. Prompt-pipeline coverage — is every governance/role/doc-gen prompt actually executed each session?

The repo contains **3 large prompt ecosystems**:

| Folder | Type | Loading mechanism | Reality |
|---|---|---|---|
| `.governance/`, `AGENTS.md`, `CLAUDE.md`, `.github/copilot-instructions.md`, `AI_EXECUTION_PROMPTS.md` | Session-start governance | Declared in `AGENTS.md` "Read order" + `CLAUDE.md` "Read order" + `.claude/hooks/session-start.sh` | ✅ **Enforced** — every AI session that runs through Claude/Cursor/Lovable loads these via hook or via the read-order block at top of `CLAUDE.md` / `AGENTS.md`. |
| `marketing/`, `BI_FRAMEWORK/`, `business_intelligence/`, `SEO/`, `crazy_innovations/`, `security-vulnerability-audit/` | Role/specialist prompts | **On-demand only.** Routed via `marketing/SYSTEM.md` (when a marketing task is detected) or invoked by user pasting the prompt. | ⚠️ **Not auto-loaded.** The agent only reads `marketing/SYSTEM.md` when a marketing-shaped task arrives. SEO/BI prompts are dormant unless explicitly invoked. This IS intentional (they'd flood context) — but it means the agent is **not** automatically running an SEO audit or BI sanity-check after each feature unless instructed. |
| `growth_strategy/`, `valuation/`, `doc creation/` | Report-toolkit generators | Carved out by `CLAUDE.md`: "running a report is a tool run, not a feature PR" | ✅ **Correct by design** — run on demand only. |
| `full-stack-e2e-prompt-ecosystem/` | E2E verification prompt library | Cataloged in `full-stack-e2e-prompt-ecosystem/README.md` + `analysis_summary.json`. | ⚠️ **Not auto-invoked.** No hook wires its modules into post-PR verification. Has 922 weak files (per its own `prompt_inventory_report.md`) that should be replaced or pruned. |

**Concrete gaps found:**
1. **No CI hook enforces governance read-order.** A human-driven AI assistant follows it (via `AGENTS.md`/`CLAUDE.md`), but an autonomous run could skip them. The `.claude/hooks/session-start.sh` exists but applies only to Claude CLI users.
2. **No automated link between feature delivery → marketing_values file.** `CLAUDE.md` mandates "Create a `marketing/marketing_values/*.md` file for every PR", but nothing enforces it; PRs land without the marketing_value file.
3. **Localization controller (`AI_PROMPTING_FOLDERSTRUCTURE/localization_controller.md`) — not auto-loaded.** Mandate is "read before any feature work involving user-facing copy" → relies on AI to self-route. Risk: untranslated strings ship.
4. **`full-stack-e2e-prompt-ecosystem/` modules are unused at PR time.** They exist as a library, not as a verification gate.
5. **Doc-generation prompts (`doc creation/SYSTEM.md`) — never invoked automatically.** They regenerate `docs/` on demand only.
6. **Versioning prompt enforcement: partial.** `versioning/*.md` files are created manually by the AI per `AI_EXECUTION_PROMPTS.md` rule — but a release without one is not blocked by CI.

**Recommendation (separate PRs, not in this audit):**
- Add a pre-commit/CI gate: every PR that touches `src/` MUST add a file under `versioning/` AND `marketing/marketing_values/`.
- Add a localization gate: every PR that adds a key to `en.ts` MUST add the matching key to `hu.ts`.
- Wire `full-stack-e2e-prompt-ecosystem/modules/*.prompt` into a "verification checklist" auto-appended to PR descriptions.
- Demote the 922 weak prompt files (per the ecosystem's own report) — either rewrite or move to `archive/`.
