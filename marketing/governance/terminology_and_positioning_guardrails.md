# Terminology and Positioning Guardrails

## Purpose
Definitive vocabulary and positioning rules for all Effectime marketing outputs. Any AI system producing Effectime copy must check against this file before finalizing.

---

## Approved product category descriptions

Use one of the following when describing what Effectime is:

- "Workforce coordination software for growing businesses"
- "People operations platform for SMBs"
- "Leave, attendance, and coverage management system"
- "Workforce management platform for 20–300 person companies"

**Never use:**
- "HR software" alone (too generic, implies HRIS market we don't fully play in)
- "All-in-one HR platform" (overpromises and undersells specificity)
- "Enterprise HR solution" (wrong audience signal)
- "People management app" (too lightweight sounding)

## Approved terminology

| Concept | Use this | Not this |
|---------|----------|----------|
| Employee time off | Leave management | Vacation management, PTO tracking |
| Who needs to be at work | Coverage requirements / coverage rules | Staffing requirements (too formal) |
| Multiple offices | Multi-site management | Multi-location (both ok but "site" preferred) |
| Daily hours worked | Time & attendance | Timekeeping (too old-fashioned) |
| Scheduled work | Shift assignments, coverage | "Rostering" (only for UK/AU markets) |
| Employee record | Member profile | Staff record, personnel file |
| Admin role | Workspace admin, HR manager | Superuser, admin user |
| Automated HR process | HR workflow | Automation (too generic) |

## Banned / avoid list

These words and phrases are prohibited across all Effectime marketing outputs unless specifically justified:

| Term | Why banned |
|------|-----------|
| "Revolutionary" / "revolutionize" | Startup cliché; no meaning |
| "Game-changer" | Overused; zero credibility |
| "Seamless" | Means nothing; every product claims this |
| "Powerful" | Vague; say what the power actually delivers |
| "Cutting-edge" | Cliché; sounds like marketing speak |
| "Robust" | Corporate jargon with no meaning |
| "World-class" | Unverifiable claim |
| "Disruptive" | Overused in SaaS; no longer meaningful |
| "AI-powered" | Only use if specific AI functionality is named and accurate |
| "Next-generation" | Meaningless without context |
| "Leverage" (as a verb) | Corporate jargon; use "use" |
| "Synergy" | Never |
| "Paradigm shift" | Never |
| "Empower" | Overused; say what specifically changes for the user |

## Competitor brand mentions

**Policy:** Do not name competitors unprompted in outbound content. Use generic alternatives:
- "generic HRIS tools" instead of naming BambooHR
- "manual spreadsheet-based approaches" instead of "Excel"
- "enterprise HR suites" instead of Workday/SAP

**Exception:** Comparison pages (marketing/website/comparison_pages_masterprompt.md) — these may name competitors directly but must follow the neutral-tone guidelines in that file.

---

## Claim substantiation requirements

A claim may only be made in marketing if it meets one of these criteria:
1. **Verified by customer evidence** — documented in a case study or customer interview
2. **Verifiable from the product itself** — a feature that demonstrably does what the claim says
3. **Industry-standard benchmark** — cite the source
4. **Qualified language** — "up to X" or "can reduce" instead of absolute claims

**Never make:**
- Unqualified ROI claims ("saves 80% of HR time") without evidence
- Compliance guarantees ("fully GDPR compliant" without legal review)
- Availability or uptime claims without SLA backing
- Integration claims without verifying the integration works end-to-end
