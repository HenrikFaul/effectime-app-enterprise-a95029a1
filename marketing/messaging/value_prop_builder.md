# Value Proposition Builder

## Purpose
Use this file to construct and validate value propositions for Effectime across different audience segments, use cases, and campaign contexts. A value proposition is not a tagline. It is a specific, believable, differentiated claim about the value Effectime delivers to a defined buyer in a defined situation. Use the formulas here, then run each draft through the validation checklist.

---

## Value Proposition Formulas

### Formula 1 — The Classic Structure
**[Effectime] helps [target buyer] [achieve outcome] without [pain / trade-off they currently accept].**

Examples:
- "Effectime helps HR managers at 50–150 person companies run a clean leave and attendance process without spending 3 hours a week on manual reconciliation."
- "Effectime helps operations managers see whether their team is covered next week without waiting for a site manager to call them with bad news."
- "Effectime helps founders of growing companies stay in control of workforce coordination without being personally involved in every leave approval."

---

### Formula 2 — The Contrast Structure
**Unlike [current approach or competitor], Effectime [specific differentiator] so that [business outcome].**

Examples:
- "Unlike an Excel leave tracker, Effectime flags coverage problems before you approve leave — so you're never accidentally understaffed."
- "Unlike generic HR software built for large enterprises, Effectime is set up and running in 2–3 weeks — so a 60-person company doesn't need an IT project to get started."
- "Unlike Jira alone, Effectime connects real HR absence data to your sprint planning — so you stop assigning work to people who won't be there."

---

### Formula 3 — The JTBD Structure
**When [situation / trigger], [target buyer] uses Effectime to [job to be done] so they can [desired outcome].**

Examples:
- "When a manager texts a leave request on a Friday, HR managers use Effectime to turn that into a tracked, approved record — so nothing gets lost in the chat."
- "When planning next quarter's project capacity, operations directors use Effectime to see real availability — so project commitments don't get blown by absences nobody logged."
- "When preparing for a labor compliance audit, founders use Effectime to pull full attendance and leave records — so the data is there without a week of manual retrieval."

---

### Formula 4 — The Outcome-Led Pitch
**[Specific result] in [time frame], without [pain / requirement they fear].**

Examples:
- "Attendance data that closes payroll in minutes, not hours — without a single spreadsheet."
- "Full leave visibility across all your sites, without asking each manager to send you a status update."
- "A compliance-ready attendance record by default — without changing how your team clocks in and out."

## Value Propositions by Persona

### For the HR Generalist
**Core VP**: Effectime helps HR managers at growing companies replace the patchwork of spreadsheets, chat threads, and manual processes with one system that tracks leave, attendance, and coverage — so they spend less time coordinating and more time doing real HR work.

**Supporting claims**:
- Leave approvals processed through one channel, with automatic calendar and coverage update
- Attendance reconciliation that takes minutes, not hours
- Coverage rules that catch problems before they become incidents
- Month-end reporting that runs itself

---

### For the Operations Manager
**Core VP**: Effectime gives operations managers a live view of team availability, coverage status, and resource capacity — so they stop discovering workforce problems after they've already become operational failures.

**Supporting claims**:
- See coverage gaps before approving leave
- Resource planning grounded in actual availability, not assumptions
- Jira integration so project planning reflects who's actually there
- Multi-site dashboard for distributed operations leaders

---

### For the Founder / Owner
**Core VP**: Effectime lets founders and owners stop being personally involved in workforce coordination by giving them visibility without micromanagement — and giving the business the records it needs for compliance and growth.

**Supporting claims**:
- Remove yourself from the leave approval loop without losing oversight
- Audit-ready attendance records without manual effort
- Scales with the company — from 30 to 200 employees without changing tools
- Up and running in weeks, not months

---

### For the Tech-Forward SMB (Jira User)
**Core VP**: Effectime connects your HR and attendance data to your Jira project workflow — so sprint capacity planning reflects who's actually available, and leave approvals don't create invisible planning surprises.

**Supporting claims**:
- Leave data visible inside Jira sprint context
- Absence flags in capacity planning views
- Works alongside your existing Jira setup — not a replacement
- Clean API-based integration, no manual sync

## Validation Checklist

Before finalizing any value proposition, verify:

**1. Does it address a real, documented pain?**
- Is this pain confirmed by customer interviews, JTBD research, or review data?
- Would the target persona say "yes, that's exactly my problem"?

**2. Is it specific enough to be believable?**
- Does it name a specific outcome (not a vague improvement)?
- Could it apply to any SaaS product, or is it specific to workforce coordination?

**3. Is it provable or demonstrable?**
- Can we show this outcome in a demo, case study, or trial experience?
- Is there a customer quote that validates this claim?

**4. Is it differentiated?**
- Does any direct competitor claim the same thing in the same way?
- If yes, is our delivery of the outcome meaningfully better?

**5. Is it relevant to the buying context?**
- Is the persona in a situation where this pain is currently active?
- Does the value prop speak to the trigger event that brought them to us?

## Value Prop Testing Protocol
- Write 3 headline variants based on the VP for a landing page or ad
- Run the headline against 50–100 impressions minimum before making conclusions
- Measure: click-through rate (for ads), conversion rate (for landing pages), time-on-page + CTA click (for website sections)
- Winning VP becomes the primary claim; losing VPs become secondary or dropped
- Document results in `analytics/experiment_backlog_masterprompt.md`
