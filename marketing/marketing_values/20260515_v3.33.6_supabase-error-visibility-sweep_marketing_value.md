# Marketing Value — v3.33.6: Supabase Error-Visibility Sweep

## Feature name and version
Silent-failure elimination across components, hooks, and edge functions — v3.33.6

## One-sentence user-language description
HR and operations managers can now trust that when a data operation fails (network,
permissions, database), they will see an error rather than silently receiving an
empty screen or incomplete data.

## Problem it replaces
Multiple places in the app silently swallowed database errors — leave calendar,
approval inbox, audit log, role permissions, and report exports would show empty
results instead of error states when Supabase returned a failure. Users had no way
to know data was missing; admins had no operational signal in logs.

## Primary beneficiary persona(s)
- **HR/Operations Manager** — approval inbox and leave calendar are mission-critical;
  silent failures cause missed requests and incorrect headcount planning.
- **Workspace Admin** — audit log completeness and role-permission reliability
  underpin compliance and access control.
- **Platform Admin** — holiday sync and report execution errors are now observable
  in edge function logs.

## Marketing-ready claim
"Effectime fails loudly, not silently — every data error is logged, surfaced, and
recoverable, never hidden behind an empty list."

## Claim types enabled
- Reliability / observability
- Trust (errors are visible, not swallowed)
- Operational confidence (admins can investigate issues)

## Content angles
1. **"Why empty screens are worse than error messages"** — a post about why silent
   failures in HR software are more dangerous than visible errors.
2. **"Observable by default"** — our commitment to structured error logging at every
   data boundary.

## Funnel stage
Retention / trust (reassures existing enterprise customers that the platform
has strong operational visibility).
