# Marketing Value — v3.42.5: Pending-Notification Guard

## Feature name and version
Duplicate-notification prevention for open shifts — v3.42.5

## One-sentence user-language description
Effectime now automatically prevents managers from accidentally sending the same shift alert twice to the same employee — anyone who's already waiting to respond appears clearly locked in the candidate list.

## Problem it replaces
Managers could unknowingly re-broadcast an open shift to the same people who were already awaiting the first notification, creating confusion, distrust, and notification fatigue among staff.

## Primary beneficiary persona(s)
- **Operations Manager / HR Manager** posting multiple open shift requests in quick succession
- **Employees** who no longer receive duplicate or redundant shift alerts

## Marketing-ready claim
*"Effectime remembers who you've already asked — pending candidates are clearly locked until they respond, so you never spam your team with duplicate shift alerts."*

## Claim types enabled
- **Intelligence** — the system tracks notification state automatically, no manual tracking needed
- **Trust** — employees receive clean, non-duplicate communications
- **Reliability** — guards against an entire class of operator error

## Content angles
1. **Product update:** "Open shift notifications now have memory — if someone is already waiting to respond, they appear locked in the candidate list for any subsequent posting."
2. **Support deflection:** Eliminates inbound tickets from employees asking "why did I get the same shift alert twice?"

## Funnel stage
- **Retention** — reduces workflow friction and notification noise, improving both manager and employee satisfaction
