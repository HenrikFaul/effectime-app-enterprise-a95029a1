# Marketing Value — v3.33.5: Data Integrity & Multi-Language Calendar Fix

## Feature name and version
Edge-function data integrity hardening + LeaveCalendar localization — v3.33.5

## One-sentence user-language description
Your GDPR data exports now correctly include every attendance and wellbeing record,
the leave calendar displays properly in all 8 supported languages, and payroll period
locking is now safe against race conditions.

## Problem it replaces
- HR managers running GDPR subject-access exports received incomplete data (attendance
  and wellbeing records were silently missing due to a wrong database column).
- Non-English-speaking teams (German, Czech, Slovak, Polish, Romanian) saw Hungarian
  text in the leave calendar weekday headers and team-filter dropdown.
- A payroll administrator locking a period could be impersonated via a tampered API
  request; concurrent lock attempts could corrupt the period status.

## Primary beneficiary persona(s)
- **HR/Compliance Manager** — GDPR export completeness is a legal obligation; silent
  data loss in subject-access responses creates audit risk.
- **Operations Manager** — Payroll period lock integrity protects payroll runs from
  accidental double-locking or unauthorized attribution.
- **International Team Leads / Members** — Correct language in the leave calendar is
  table-stakes UX for non-Hungarian workspaces.

## Marketing-ready claim
"Effectime's GDPR data export is complete and legally defensible — every attendance
and wellbeing record is included, in every supported language."

## Claim types enabled
- Compliance / legal defensibility (GDPR export correctness)

- Trust / security (payroll lock attribution tied to verified identity)
