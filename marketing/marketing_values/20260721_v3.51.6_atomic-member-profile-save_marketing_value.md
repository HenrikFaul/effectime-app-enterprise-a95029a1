# Marketing Value — v3.51.6 Reliable member profiles on web and mobile

**Feature/version:** Atomic workspace-member profile editing — v3.51.6

**Delivery status:** source candidate only. The local PostgreSQL and client
contracts pass, but hosted CI, merge, restored staging, database apply and
production deployment are not complete. This document records potential value;
it is not evidence that the capability is live.

## User-language outcome

Effectime can keep a team member's role, workplace, working hours and allocation
changes together across web, Android and iOS—either the whole update succeeds,
or existing data stays intact and the user is asked to reload.

## Customer value

- **Data integrity:** one save no longer consists of independent partial commits.
- **Concurrency safety:** simultaneous browser or mobile edits do not silently
  overwrite a newer profile.
- **Tenant safety:** offices, allocations and permissions are rechecked on the
  server for the selected workspace.
- **Consistent platforms:** web, Android and iOS use the same Supabase source and
  the same validated business contract.
- **Recoverable UX:** load errors and conflicts are explicit, localized and do
  not turn missing data into an editable empty profile.
- **Auditability:** a successful change produces one minimal, transaction-bound
  audit receipt without copying profile values into browser-visible history.

## Marketing-ready claim after production acceptance

> Update member roles, locations, working hours and allocations with one
> conflict-safe workflow on web and mobile—without partial saves or a separate
> mobile data store.

## Required caveats

- Do not claim this is live until database-first rollout and live release-SHA
  verification pass.
- Do not claim zero data-loss risk, regulatory certification or store
  availability from source/CI evidence alone.
- Historical invalid allocations may require reviewed remediation before the
  migration can be applied; the candidate deliberately refuses silent cleanup.
- The separate legacy role-allocation manager still needs an atomic total/
  priority enforcement package before broad marketing of allocation editing.
- Android/iOS source parity does not mean signed App Store/Play Store release.
