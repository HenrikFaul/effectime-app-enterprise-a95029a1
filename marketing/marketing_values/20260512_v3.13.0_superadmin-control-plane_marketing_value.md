# Marketing Value — v3.13.0 Superadmin / Platform Control Plane

## Feature name and version
Superadmin Platform Control Plane — v3.13.0

## One-sentence user-language description
Effectime now ships with a dedicated platform control plane giving operators full visibility and control over every workspace, feature flag, scheduled job, locale, and email queue — from a single, security-gated admin surface.

## Problem it replaces
Platform operators had no single place to see the health of all workspaces at once, toggle feature flags without a code deploy, inspect scheduled jobs, or monitor the email queue. Diagnosing issues required direct database access or ad-hoc queries — slowing incident response and making self-service platform management impossible.

## Primary beneficiary personas
- **Platform Operator / SaaS Admin** — single pane of glass for all workspaces, jobs, and platform health without touching a database
- **CTO / Engineering Lead** — feature flags enable safe rollouts and instant rollback without deploys; visibility into pg_cron job status
- **Customer Success / Support** — archive or put a workspace into recovery mode in one click, no SQL required
- **Data Protection / Compliance Officer** — locale registry shows exactly which locales are enabled at the platform level, supporting GDPR jurisdiction mapping

## Marketing-ready claim
"Effectime gives platform operators a full control plane out of the box: workspace health, feature flag toggles, scheduled job management, and locale registry — all in one security-gated admin surface, with no database access required."

## Claim types enabled
- **Operational visibility** — real-time overview of all workspaces, users, queue depth, and platform flags
- **Platform reliability** — trigger, inspect, and monitor scheduled jobs from the UI; no cron log diving
- **Enterprise control** — archive, recover, or delete workspaces with audit trail; feature flag rollouts without deploys
- **Security** — admin-only endpoint enforced at backend (not UI-only); service role key never exposed to client; allowlisted function triggers
- **Developer experience** — feature flags reduce deploy risk; pg_cron visibility reduces ops toil
