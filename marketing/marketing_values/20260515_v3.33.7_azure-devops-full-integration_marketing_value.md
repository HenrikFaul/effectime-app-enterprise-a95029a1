# Marketing Value — v3.33.7: Azure DevOps Full Integration

## Feature name and version
First-class Azure DevOps integration — v3.33.7

## One-sentence user-language description
Engineering teams using Azure DevOps can now sync project config, browse work items, and edit them inline inside Effectime — with the same depth of integration that Jira users have always enjoyed.

## Problem it replaces
Azure DevOps support existed in name only: it could list work items via WIQL, but
had no inline editor, no work item type discovery, no iteration path discovery,
no state transitions, and no user/identity search. Teams using Azure DevOps had
to leave Effectime to make any change. This release eliminates that gap.

## Primary beneficiary persona(s)
- **Engineering Manager (Azure DevOps shop)** — can now plan sprints, assign
  work items, and track state transitions without leaving the Effectime workforce
  planning view.
- **Resource / Program Manager** — iteration path discovery lets them map ADO
  sprint structure to Effectime capacity planning; no manual data entry.
- **HR / Ops Manager** — can see which sprint employees are assigned to and
  cross-reference with leave requests directly in the agile panel.

## Marketing-ready claim
"Whether your team runs on Jira or Azure DevOps, Effectime gives you the same
depth of agile integration — work item editing, state transitions, identity
search, and capacity planning all in one place."

## Claim types enabled
- Platform parity (Jira ↔ Azure DevOps equivalence)
- Enterprise depth (metadata-driven, not hardcoded)
- Workflow unification (no context-switching between tools)
- Microsoft ecosystem support

## Content angles
1. **"Two task managers, one workforce platform"** — position Effectime as the
   neutral hub that connects both Atlassian and Microsoft toolchains to HR data.
2. **"No more tab-switching"** — highlight inline work item editing and state
   transitions without leaving Effectime.
3. **"Sprint-aware leave planning"** — show how iteration path discovery lets
   managers see leave impact on sprint capacity.

## Funnel stage
Acquisition / expansion:
- Acquires net-new enterprise prospects in Microsoft-stack shops.
- Expands existing Jira-only customers to cross-team use when some teams use ADO.
