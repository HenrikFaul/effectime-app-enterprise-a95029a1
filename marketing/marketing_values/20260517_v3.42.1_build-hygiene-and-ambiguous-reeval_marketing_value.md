# v3.42.1 — Build hygiene + ambiguous-items re-eval

**Released:** 2026-05-17
**Internal-only release** — no user-facing features changed.

## Problem solved (in plain user language)

None directly user-visible. This release closes a small but growing
backlog of TypeScript build warnings that, if left unattended, would
eventually break preview deploys and slow down every future feature
PR.

## Personas impacted

| Persona | How they benefit |
|---------|------------------|
| Engineering | Build pipeline is green again; future PRs no longer carry pre-existing error noise. |
| Operator / IT lead | The DB audit governance trail now has a re-evaluation marker confirming nothing was deleted between cycles. |

## Marketing claims unlocked

- **"Engineering hygiene cadence"** — we run periodic build-error
  cleanups and database-audit re-evaluations as a standing practice,
  not only when something breaks.
- **"No-regression discipline"** — even cleanup work follows a strict
  "create the missing dependency, never delete the consumer" rule,
  documented per release.
