## User-language problem solved

Not a user-visible feature — a data-integrity release.

## Primary personas
- **Compliance lead** — wants every privileged platform action to leave
  an immutable audit row (GDPR Art. 5, ISO 27001 A.12.4).
- **CFO / billing owner** — wants tier-id changes to be deliberate and
  traceable, never silent.
- **IT manager** — wants tier-locked features to stay locked, even when
  the underlying tier mapping changes.

## Marketing-ready claim
"Effectime enforces tier-immutability and audit trail at the database
level. Tier changes can only happen through an admin-approved,
audit-logged operation. No silent fallback. No undocumented bypass."

## Claim types enabled
- **Compliance** — append-only platform audit trail covers all
  superadmin actions.
- **Reliability** — DB-level invariants (triggers, CHECK constraints,
  RLS filtering) replace convention-only governance.
- **Transparency** — partial failures in scheduled reports and demo
  cleanup are now surfaced instead of hidden.
