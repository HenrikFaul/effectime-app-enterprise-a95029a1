# Marketing Value — v3.47.2: E2E Verification + DB Fixes

**Date:** 2026-05-23
**Version:** v3.47.2
**Feature slug:** e2e-db-fixes

---

## User-language problem solved

"We integrated the Effectime Embed SDK into our CRM and tried to revoke a token that a developer accidentally shared — but nothing happened. No error, no confirmation. We had no idea if the revocation worked."

This release fixes the silent revocation failure and closes a DB migration gap that would have caused CRM integrations calling the new availability and headcount APIs to receive "function not found" errors.

---

## Personas who benefit

- **Enterprise IT / Integration Lead** — now gets an explicit error when a token revocation fails (wrong permissions, non-existent token), instead of silent no-op
- **Workspace Admin / Security-conscious owner** — can trust that `Revoke` in the embed token UI either succeeds or shows an error — no ambiguity
- **CRM developer using new RPCs** — `check_member_availability` and `get_team_headcount` now exist in production (were missing despite being in the codebase)
