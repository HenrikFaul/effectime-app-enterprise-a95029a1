## User-language problem solved

Internal-only release. End users see no UI change.

For platform operators (super-admins editing the Feature Tiers tab), the
"page" grouping in the Feature Tiers tree is now consistent and
disambiguated: 31 workspace features that previously all collapsed into
the workspace root now each have their own canonical sub-path; 111
features still labelled with the retired `/app/...` shape have been
re-labelled to the current `/w/:workspaceId/...` shape.
