# Marketing Value: v3.41.7 — Open Shift Candidates Now Filtered by Position

**Date:** 2026-05-18
**Version:** v3.41.7
**Feature slug:** open-shift-candidate-position-filter
**Type:** Data Integrity / UX Correctness

---

## User-Language Problem Solved

When a manager opened a shift for a specific position — say, "Optometrist" — the system
suggested candidates from completely unrelated roles (Backend Developer, Operations Lead)
while simultaneously missing one of the actual optometrists on the team.

The result: managers either sent shift notifications to the wrong people, or had to manually
hunt for the right person in a separate view. This eroded trust in the scheduling tool and
created extra work on every open-shift post.

The fix ensures that candidate suggestions always and only contain people whose position
matches the open shift. If two people hold the "Optometrist" role, both appear. No one
else does.

---

## Which Personas Benefit

- **HR Manager / Shift Scheduler** — Candidate shortlist is immediately trustworthy.
  No need to cross-reference a capacity view or a member roster. Click a position,
  see the right people, send the notification.
- **Enterprise Admin / Platform Owner** — Demonstrates that the platform's data model
  (positions → people) is honoured end-to-end, not just in reports.
- **Compliance / Audit** — Open-shift records now correctly reflect position-qualified
  candidates, not an arbitrary availability ranking.
