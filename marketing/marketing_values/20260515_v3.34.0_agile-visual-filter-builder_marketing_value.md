## Problem solved (user language)

Previously, filtering your Azure DevOps or Jira backlog in Effectime required writing WIQL or JQL query language — a technical barrier that made the feature inaccessible to non-developers (project managers, HR leads, team leads). There was also no way to control which work item fields appeared on the board.

## Solution delivered

A fully visual, click-based backlog filter builder:
- **Select work item types** via checkboxes (Bug, Task, User Story, Epic…)
- **Filter by state** (Active, New, Resolved…) via checkboxes
- **Filter by assignee** — choose from your actual project members, not free-text
- **Filter by iteration path** — dropdown from your real sprint/iteration tree
- **Date range picker** for creation date
- **Title text search**
- Active filter badges visible at a glance, each removable with one click

Plus a new "Fields" selection panel: choose which work item fields appear on your board via checkboxes and save with one click. Your selection persists per integration.

Both Jira Cloud and Azure DevOps are fully supported.
