# v3.15.0 — Feature Tiering completion — Marketing Value

## Feature name and version

Feature Tiering — End-to-end activation (v3.15.0), follow-up to the v3.13.x
platform tiering rollout.

## One-sentence user-language description

Every customer now sees only the modules included in their subscription
(Freemium / Pro / Enterprise), with premium areas blocked behind a clear
upgrade message and an audit log of every plan change kept for compliance.

## Problem it replaces

Until now the tier system was effectively decorative — features were
catalogued and a Superadmin could toggle them, but the actual app never read
the toggles, so every workspace saw every module regardless of plan. That
meant: (a) Freemium users could open Enterprise-only screens, (b) operators
couldn't confidently sell tier-based pricing, and (c) no audit trail existed
when tier mappings were changed.

## Primary beneficiary personas

- **Platform owner / SaaS operator** — can now sell three real plans with
  confidence that the UI enforces what they sold.
- **Enterprise buyer / CIO** — sees the audit log for every tier change
  (SOC 2 / ISO 27001 evidence).
- **Sales / customer success** — the demo workspace creator now produces a
  tier-appropriate dataset (Freemium = 6 members, Pro = 12 members, Enterprise
  = full catalog), so demos match the customer's intended plan.
- **End user / admin** — gets a clear "this feature is not in your plan"
  notice instead of a broken or empty screen when they hit a locked module.

## Marketing-ready claim

"Sell three real plans, not a feature list. Effectime now enforces every
subscription tier in the UI, ships an immutable audit log of plan changes,
and generates demo workspaces sized to match the customer's plan."

## Claim types enabled

- **Visibility** — users always know which features are in their plan; no
  more "click and see if it works."
- **Compliance** — immutable `platform_audit_events` table records every
  tier/addon/routing change with actor, target, and timestamp.
- **Coordination** — Sales can run a Freemium demo and an Enterprise demo
  side-by-side and the data volume / module surface differs realistically.
- **Cost reduction** — fewer support tickets from Freemium users trying to
  access Enterprise features.
- **Conversion** — premium tabs now show a localized "upgrade to unlock"
  card instead of disappearing silently, creating a natural upsell hook.
